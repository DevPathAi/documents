# 08. 스토리 보드

> **목적**: 주요 페르소나의 End-to-End 여정과 핵심 유저 스토리
> **작성 형식**: "사용자로서 ~를 하고 싶다. 왜냐하면 ~이기 때문이다." + AC (Acceptance Criteria)

---

## 1. 페르소나

### P1. 지수 — 부트캠프 3개월차 (24세)
- 상태: 국비 부트캠프에서 Spring Boot 배우는 중, 취업 준비
- 고민: **"무엇을 얼마나 알아야 취업할 수 있는지 모르겠다"**. 학원 커리큘럼 끝난 후 막막함
- 기대: 내 현재 실력 기반 3개월 로드맵 + 매일 할 일 알려주기
- 기기: Android (Galaxy S22)

### P2. 준호 — 이직 준비 주니어 (27세)
- 상태: 3년차 주니어 백엔드. 현재 회사에서 성장 정체 느낌
- 고민: **"내 약점이 뭔지 정확히 모르고, 이직 포트폴리오 방향을 못 잡음"**. Anki + Notion + 유튜브 파편화
- 기대: GitHub 활동 기반 객관적 진단 + 약점 저격 학습
- 기기: iPhone 14 + MacBook

### P3. 수민 — 시니어 실무 강화 (31세)
- 상태: 7년차, 스타트업에서 MSA 도입 프로젝트 리드 중
- 고민: **"내 코드에 실무급 피드백을 받을 곳이 없다. 팀에 공유할 학습 자산이 필요하다"**
- 기대: 실무급 AI 코드 리뷰로 본인 개선 + 팀 학습 레퍼런스 확보
- 기기: iPhone 15 Pro + MacBook

### P4. 태윤 — 플랫폼 운영자 (28세)
- 상태: DevPath 운영/SRE 담당
- 고민: Sandbox 악용, Claude 한도 폭주가 가장 무서움
- 기대: FinOps + Sandbox 보안 상시 가시성

## 2. 학습자 Epic

### Epic L1. 진입 — 0초~8분 (🎉 1st Aha)

**Story L1.1 — 비회원 체험으로 신뢰 먼저**
> 방문자로서 **가입하기 전에 제품이 쓸만한지 5분 안에** 확인하고 싶다. 왜냐하면 낯선 서비스에 처음부터 GitHub 권한을 주기는 부담스럽기 때문이다.

**AC:**
- [ ] 랜딩 주 CTA → 비회원 진단 즉시 시작
- [ ] 15문항 적응형, 5분 내 완료 가능
- [ ] 결과 페이지 진입 직전에 로그인 요구 (게이트)
- [ ] 결과는 로그인 후 "원래 있던 것처럼" 표시 (게스트 id 이관)

**화면**: SCR-W-LAND-001, SCR-W-ONB-002

~~**Story L1.2 — AR 데모로 "이건 다르다" 각인** (v2.0 — 현재 비활성)~~

**Story L1.3 — GitHub로 즉시 개인화**
> 개발자로서 **이미 공개된 내 GitHub 활동을 보고 맞춤 추천**을 받고 싶다. 왜냐하면 진단 테스트만으론 진짜 실력이 드러나지 않기 때문이다.

**AC:**
- [ ] GitHub 로그인이 주요 OAuth 선택지
- [ ] 백그라운드로 리포·언어·커밋 수집
- [ ] 진행 표시: "GitHub 분석 완료 →" 배지
- [ ] 경로 생성 시 Claude 프롬프트에 GitHub 요약 삽입

**화면**: SCR-W-AUTH-001, SCR-W-ONB-003

**Story L1.4 — 🎉 "왜 이 순서"가 설득되는 경로**
> 학습자로서 **AI가 짜준 경로가 왜 이 순서인지** 알고 싶다. 왜냐하면 "그냥 AI가 만든 것"이면 믿고 따르기 어렵기 때문이다.

**AC:**
- [ ] 각 주차마다 1~2줄의 rationale 코멘트
- [ ] 진단 결과의 약점과 경로 초반이 명확히 연결
- [ ] "3주 후 당신은 ..." 역량 목표 표시
- [ ] CTA는 "오늘의 학습 시작" 단 하나

**화면**: SCR-W-PATH-001

---

### Epic L2. 첫 학습

**Story L2.1 — 콘텐츠 안에서 바로 실행**
> 학습자로서 **긴 글을 읽다가 코드를 바로 실행**하고 싶다. 왜냐하면 IDE를 켜서 복붙하는 순간 흐름이 끊기기 때문이다.

**AC:**
- [ ] 코드 블록마다 "실행해보기" 버튼
- [ ] 500ms 내 Sandbox 팝오버
- [ ] 수정 후 재실행 즉시 가능

**화면**: SCR-W-CNT-001, SCR-W-SBX-001

---

### Epic L3. 실습 — 🎉 2nd Aha (실무급 피드백)

**Story L3.1 — 단순 정오답이 아닌 실무 리뷰**
> 학습자로서 **"통과" 이상의 피드백**을 받고 싶다. 왜냐하면 현업에 가면 "왜 이렇게 짰냐"를 묻기 때문이다.

**AC:**
- [ ] 리뷰에 강점·개선·보안 3분류
- [ ] 라인 번호 구체적 지시
- [ ] "LazyInitializationException 가능성" 수준의 현업 지적
- [ ] 👍👎 피드백 수집 (품질 A/B)

**화면**: SCR-W-REV-001

**Story L3.2 — 내 맥락을 아는 멘토**
> 학습자로서 **지금 내가 뭐 하고 있는지 아는 AI**에게 질문하고 싶다. 왜냐하면 일반 ChatGPT는 매번 맥락을 다시 써야 하기 때문이다.

**AC:**
- [ ] 세션 생성 시 현재 콘텐츠 + 최근 5 Sandbox 자동 주입
- [ ] 응답에 참고 자료 (콘텐츠/sandbox 링크) 명시
- [ ] 실패한 테스트 이름을 답변에 인용

**화면**: SCR-W-MEN-001

---

### Epic L4. 습관화 — 커뮤니티 + 리텐션 루프

**Story L4.1 — 혼자 공부하는 외로움 줄이기**
> 학습자로서 **같은 주제를 공부하는 사람이 있다**는 것을 알고 싶다.

**AC:**
- [ ] "같은 주 23명" 배지
- [ ] 피어 매칭 Q&A 노출
- [ ] 질문 작성 → 매칭 피어에게 알림

**화면**: SCR-W-COM-001, SCR-W-COM-002

**Story L4.4 — 내 학습 맥락을 아는 답변자**
> 학습자로서 **답변자가 내가 어느 수준·어느 콘텐츠에서 막혔는지** 이미 알고 답해주면 좋겠다. 왜냐하면 매번 "저는 주니어고..."로 시작하는 배경 설명이 지친다.

**AC:**
- [ ] 질문 작성 시 학습 맥락 자동 첨부 토글 (기본 on)
- [ ] 맥락 항목 개별 on/off + 미리보기
- [ ] 답변자 페이지에 "질문자 맥락" 패널 노출
- [ ] 공개 범위: 답변자만 / 공개

**Story L4.5 — 답 기다릴 필요 없는 Q&A**
> 학습자로서 **질문 즉시 AI 초안이 달려** 빠르게 풀리거나, 부족하면 인간 답변으로 올라가면 좋겠다.

**AC:**
- [ ] Q&A 작성 후 3~5초 내 AI 답변 자동 생성 (뱃지 "🤖 AI 초안")
- [ ] "도움됨/불충분" 평가
- [ ] 불충분 시 커뮤니티 에스컬레이션 CTA + 현상금 옵션(평판 1,000+)
- [ ] 유사 질문 유사도 80% 이상이면 기존 답변 안내

**Story L4.6 — 쌓이는 평판과 태그 전문성**
> 학습자로서 **내가 답변할수록 특정 기술 스택에서 전문가로 인정받는** 느낌을 원한다.

**AC:**
- [ ] Bronze 배지 9종 자동 수여 + 알림
- [ ] 프로필에 총 평판 + 태그별 평판 공개
- [ ] 평판 15/125/500 도달 시 권한 언락 안내
- [ ] 일일 +40 상한 + 7일 신규 투표 제한 + sockpuppet 방지

**Story L4.7 — 내 프로젝트 피드백 받기**
> 사이드 프로젝트를 만든 학습자로서 **실제 현직자 피드백**을 받고 싶다.

**AC:**
- [ ] 프로젝트 공유 게시판 (GitHub URL 필수)
- [ ] 스타 기능, 50개 이상 시 "주목받는 프로젝트" 뱃지
- [ ] "코드 리뷰 요청" 버튼 → Q&A 자동 연동

### Story L4.8 -- 자유게시판에서 커리어 고민 나누기 (CEO 리뷰 보강)

> 학습자로서 **기술 질문 외에도 커리어/이직/회사 고민**을 나눌 곳이 있으면 좋겠다.

**AC:**
- [ ] 자유게시판 카테고리 6종 (커리어/취업/회사생활/잡담/후기/뉴스 공유)
- [ ] 좋아요 50+ 시 인기글 배지 + 글쓴이 평판 +20
- [ ] Markdown 에디터 지원

### Story L4.9 -- Founding Contributor로 초기 커뮤니티 세우기 (CEO 리뷰 보강)

> 경력 개발자로서 **새로운 커뮤니티를 처음부터 만드는 데 참여**하고 싶다.

**AC:**
- [ ] 프로필에 "Founding Contributor" 영구 배지
- [ ] 초기 평판 500 부여 (편집 제안, downvote 등 권한 해금)
- [ ] 답변 1개당 5,000원 소액 페이 (3개월, 월 상한 30만원)
- [ ] Pro 플랜 6개월 무료
- [ ] Go/No-Go: 30명 이상 GO, 10명 미만 NO-GO

### Story L4.10 -- 모더레이션으로 건강한 커뮤니티 유지 (CEO 리뷰 보강)

> 운영자로서 **AI 자동 필터 + 신뢰 사용자 위임**으로 모더레이션 부담을 줄이고 싶다.

**AC:**
- [ ] AI 모더레이션 4단계 심각도 자동 분류 (LOW/MED/HIGH/CRITICAL)
- [ ] AI 실패 시 규칙 기반 필터 fallback (Eng 리뷰 결정)
- [ ] 관리자 모더레이션 큐 (SCR-A-COM-001)
- [ ] 제재 단계: 경고 → 7일 → 30일 → 영구
- [ ] 이의제기 24~72h 처리
- [ ] CRITICAL: 즉시 비공개 + 관리자 확인 후 계정 정지 (자동 정지 금지)

**Story L4.2 — 내 리듬에 맞는 알림**
> 학습자로서 **약속한 시간에만 알림**이 오면 좋겠다. 왜냐하면 랜덤 푸시는 짜증나기 때문이다.

**AC:**
- [ ] 온보딩에서 선호 시간대 수집
- [ ] 정확히 그 시간대에 "오늘의 15분"
- [ ] 일요일 저녁 주간 리포트
- [ ] 3일 미접속 시 AI가 난이도 재조정 제안

**Story L4.3 — 성장을 눈으로 보기**
> 학습자로서 **이번 주 내가 성장한 것 3가지**를 보고 싶다.

**AC:**
- [ ] 주간 리포트: 신규 개념·해결 과제·배지
- [ ] 스트릭 시각화 (GitHub 스타일 히트맵)
- [ ] 역량 레이더 차트 (진단 대비 변화)

**화면**: SCR-W-DASH-001

## 6. 핵심 내러티브 — DevPath AI만이 가능한 경험

```text
1. 가입 5분 만에 "내 GitHub + 진단"이 반영된 3개월 로드맵을 받는 경험
   → 다른 강의 플랫폼: 수천 강의 중 골라야 함

2. Sandbox에 제출하면 "line 42에서 LazyInit 가능" 수준의 실무 리뷰를 받는 경험
   → 일반 IDE 린터: 문법만 체크

3. AI 멘토에게 물었을 때 "당신이 방금 실패한 testFetchJoin을 보면..."
   → ChatGPT: 매번 맥락 다시 제공해야

4. 학습 맥락이 자동 첨부된 Q&A에서 답변자가 내 수준을 알고 답변하는 경험
   → OKKY / 인프런 커뮤니티: 매번 배경 설명 필요

이 네 경험은 **같은 사용자 데이터 위에 쌓여서** 시간이 갈수록 정밀해진다.
이것이 DevPath AI의 존재 이유.
```
