# DevPath AI — 모두의 창업 1차 통과 이후 장기 전략

> **문서 목적**: 모두의 창업 1차 통과 (지역 예선 합격) 이후 **권역 본선 → TV 오디션 결선**까지의 90일 통합 전략. 동시에 **예비창업패키지 병행 준비**, **MVP 개발 일정**, **보육기관 활용** 등 다중 트랙 최적화.
>
> **핵심 인식**: 1차 통과는 **5,000명 풀에 진입**한 것이지 **선정 확정**이 아님. <strong>지금이 가장 위험한 시점</strong>. 이유는 "통과했다"는 안도감이 실행 속도를 떨어뜨리기 때문.
>
> **작성일**: 2026-04-23
> **문서 버전**: v1.0

## 📊 모두의 창업 단계 구조 (재정리)

### 전체 구조

| 단계 | 명칭 | 평가 방식 | 통과율(추정) | 보상 |
|:---:|------|---------|:---------:|------|
| 0 | 신청·접수 | 도전신청서 1장 | 100% (전원 접수) | 도전 경력증명서 발급 |
| **1** | **지역 예선** | **서류·아이디어 평가** | **약 5,000명 / 전체** | **✅ 통과 (현재 위치)** |
| 2 | 권역 본선 (5개 권역) | 발표·심사·멘토링 | 약 1,000~1,500명 | 보육 + 시제품 제작 지원 |
| 3 | TV 오디션 결선 | 전국 방송 | **최종 100명** | 1억~10억 사업화 자금 + 후속 투자 연계 |

### DevPath AI의 위치

- **현재**: 1차 통과 (지역 예선 합격) ✅
- **거주**: 부산 → **동남권 본선** 진출 예정
- **다음**: 보육기관 매칭 + 권역 본선 준비

### 권역 본선의 진짜 정체

권역 본선은 단순한 "다음 단계 발표"가 아닙니다. 보육기관과 매칭되어 **수 주~수 개월의 사업 고도화 기간**이 있고, 그 과정을 통해 본선 발표가 진행됩니다.

즉, 본선 진출이란 **보육기관에 입주해서 사업을 발전시키는 기회**를 얻는 것입니다. 이 기간 동안 멘토와 협업하면서 사업이 얼마나 진척되는지가 평가됩니다. **단순 발표 실력보다 실제 사업 진척도가 결정적**입니다.

## 🎯 핵심 전략 4가지 축

### 축 1: 본선 진출을 위한 차별화 강화

1차 통과자 5,000명 중 약 30%가 AI 키워드를 가졌다는 통계는 충격적입니다. 이는 본선 심사에서 "AI"는 더 이상 차별점이 아니라는 의미입니다. DevPath AI가 본선에서 살아남으려면 **"AI 기반 X"가 아닌 더 구체적이고 검증 가능한 차별화**가 필요합니다.

**전략**: 다음 3가지를 가시화

| 차별화 요소 | 강화 방향 |
|------------|----------|
| **학습 맥락 자동 첨부(LCS)** | 한국 최초임을 강조, 실제 데모 가능한 프로토타입 |
| **창업자 lived experience** | "내가 겪었고, 그래서 만든다"는 서사 일관성 |
| **이미 만든 결과물** | StockPilot, LearnFlow AI를 데모 가능한 상태로 |

### 축 2: 보육기관 매칭 최적화

본선 진출자에게는 보육기관이 매칭됩니다. 어떤 보육기관과 매칭되느냐가 **앞으로의 사업 진척에 결정적**입니다. 단순히 배정받지 말고 **선호 기관에 적극 어필**해야 합니다.

**DevPath AI 핏 보육기관 후보**:

| 기관 | 핏 점수 | 이유 |
|------|:---:|------|
| **프라이머** | ⭐⭐⭐⭐⭐ | 초기 B2C SaaS 전문, 데보시오·캐치테이블 등 유사 모델 육성 경험 |
| **퓨처플레이** | ⭐⭐⭐⭐⭐ | 기술 기반 스타트업 전문, AI/SaaS 포트폴리오 다수 |
| **소풍벤처스** | ⭐⭐⭐⭐ | 임팩트·사회적 가치 중심, "교육 격차 해소" 서사와 맞음 |
| **KAIST 창업원** | ⭐⭐⭐ | 기술 검증·R&D 강점, 학술적 깊이 |
| **부산 지역 AC** | ⭐⭐⭐⭐ | 지역 우선 배정 시 시너지, 비수도권 70% 정책 |

### 축 3: MVP 개발 일정 통합

본선 발표 시점까지 **단순 PPT가 아니라 동작하는 데모**가 있는 것과 없는 것은 결과가 완전히 다릅니다. 8주 MVP 계획을 본선 일정과 정확히 동기화해야 합니다.

**제안 일정**:

- **0~4주**: 보육기관 매칭 + 인프라 기반 + 진단 모듈 (MVP의 핵심 흐름 1개만)
- **4~8주**: AI 멘토 + 기본 Q&A + 학습 맥락 첨부 데모
- **본선 발표일**: 라이브 데모 가능한 알파 버전 확보

### 축 4: 예비창업패키지 병행

모두의 창업과 예창패는 **동시 진행 가능**한 경우가 많습니다. 단 동일 아이템으로 중복 자금은 받기 어렵습니다. 그러나 두 사업의 **단계별 인정과 평판 효과는 상호 보강**됩니다.

**병행 전략**:
- 모두의 창업 본선 진출은 예창패 심사에서 "외부 검증 완료" 신호
- 두 사업 모두 PSST 구조 사업계획서가 기반이므로 한 번 작업이 양쪽 활용
- 모두의 창업에서 멘토 매칭되면 그 멘토를 예창패 자문으로도 활용

|:---:|---------|---------|
| 3주 | 인프라 + OAuth + User Service | 발표 덱 미세 조정 |

**시나리오 A: 본선 통과 → 결선 진출**
- TV 오디션 결선 대비 (방송 친화도 강화)
- MVP 베타 100명 모집
- 30일 리텐션 40%+ 검증

**시나리오 B: 본선 탈락 → 예창패 집중**
- 본선 경험을 예창패 사업계획서에 즉시 반영
- 도전 경력증명서 활용
- 모두의 창업 2차 모집 (6월 말~7월 초) 재도전 검토

**시나리오 C: 본선 통과 + 예창패 병행**
- 두 트랙 동시 진행, 자금원 다변화
- 단 중복 수혜 제한 조항 사전 확인

## 🎬 본선 발표 차별화 전략

### 1차 통과자의 30%가 AI 보유 → 어떻게 살아남을까

#### 메시지 재구성

**Before (1차 통과 시 강조했던 것)**:
> "AI 기반 개발자 학습 플랫폼"

**After (본선용 강조)**:
> "한국 최초 학습 맥락 기반 커뮤니티" (LCS 기술 차별화)
> 
> "타겟 고객의 고통을 직접 겪은 풀스택 엔지니어가 만든다" (lived experience)
> 
> "이미 프로덕션급 시스템 2개를 단독 구축한 실행력" (증명된 역량)

이 세 가지 메시지가 **AI 키워드만 가진 다른 4,999명과의 차별점**입니다.

#### 본선 발표 덱 보강 포인트

이전 피치 덱 15장 중 보강할 부분:

| 슬라이드 | 현재 | 본선용 보강 |
|---------|------|----------|
| 6 (LCS) | 다이어그램 | **동작하는 데모 영상 30초** 추가 |
| 7 (경쟁) | 비교표 | **실제 ChatGPT vs DevPath AI 답변 비교 화면** |
| 10 (매출) | 시뮬레이션 | **MVP 베타 사용자 X명의 실제 데이터** |
| 12 (팀) | 1인 창업자 | **보육기관 매칭 결과 추가** |
| 13 (자금) | 1억 배분 | 모두의 창업 단계별 자금 활용 명시 |

#### 발표 시간 단축

본선은 5~7분 발표가 많습니다. 기존 15장을 **10~12장으로 압축**해 핵심만 남깁니다. 빠진 슬라이드는 Q&A에서 보강.

## 🤝 보육기관 활용 마스터플랜

### 매칭 전 준비

보육기관과의 첫 만남에서 "이 창업자는 진지하다"는 인상을 줘야 합니다. 다음 자료를 미리 준비:

| 자료 | 활용 |
|------|------|
| 사업계획서 PSST 풀버전 | 첫 미팅 시 공유 |
| 피치 덱 15장 | 5분 내 핵심 전달 |
| PIA 문서 | "규제 대응 준비 완료" 신호 |
| StockPilot·LearnFlow AI 데모 | 기술 역량 증명 |
| 통계 검증 가이드 | 정확성·진정성 어필 |

이 자료들이 이미 다 준비된 상태입니다. **다른 1차 통과자와의 결정적 차이점**입니다.

### 보육기관에 적극 요청할 것들

수동적으로 받기만 하지 말고, 다음을 적극 요청:

1. **포트폴리오사 소개**: 비슷한 SaaS 기업 창업자와 1:1 미팅
2. **B2B 고객 소개**: 부트캠프·중소 IT기업 대상 파일럿 연계
3. **법률 자문 연결**: 개인정보 전문 변호사 (PIA 검토용)
4. **투자자 네트워크**: 시드 라운드 사전 연결
5. **PR 기회**: 보육기관 데모데이·미디어 노출

### 보육기관 첫 미팅 체크리스트

- [ ] 본인 학력·경력을 솔직히 공개
- [ ] DevPath AI 핵심 차별화 3가지를 1분 내 전달
- [ ] 보육기관에 무엇을 기대하는지 명확히
- [ ] 매주 진척 보고 빈도·형식 합의
- [ ] 멘토와의 정기 미팅 일정 확정

## 💼 예비창업패키지 병행 전략

### 두 사업의 비교

| 항목 | 모두의 창업 | 예비창업패키지 |
|------|------------|--------------|
| **지원금** | 단계별, 결선 1위 2억 | 평균 5천만 ~ 최대 1억 |
| **평가 방식** | 오디션 (3단계) | 서류·발표 평가 |
| **기간** | 약 6개월 (단계별) | 약 10개월 (선정 후) |
| **양식** | 도전신청서 1장 → 본선 발표 | PSST 사업계획서 + 발표 |
| **보육·멘토링** | 매우 강함 (보육기관 매칭) | 보통 (주관기관 멘토) |
| **PR 효과** | 매우 강함 (TV 방송) | 보통 |
| **자금 자유도** | 사업화 자금 (낮은 제약) | 바우처 (높은 제약) |

### 병행 가능성

**가능**: 두 사업을 동시 신청하는 것은 가능하지만, 동일 아이템으로 중복 지원금을 받을 수는 없는 것이 일반적. 따라서 **단계별 진행은 가능**하나 **자금 수혜는 한쪽**.

**전략**:
1. **모두의 창업을 메인 트랙**으로 (이미 1차 통과 + 보육기관 매칭 + PR 효과)
2. **예창패는 백업 트랙**으로 (모두의 창업 본선 탈락 시 즉시 전환)
3. 두 사업의 사업계획서·발표 자료는 **공통 자산**으로 관리

### 예창패 신청 시점

모두의 창업 본선 결과 확인 후, 통과 시 → 결선 집중, 탈락 시 → 즉시 예창패 공고 신청 (보통 4월 말 ~ 5월 공고).

## 💻 MVP 개발 일정 (본선 동기화)

### 핵심 원칙

**본선 발표일까지 라이브 데모 가능한 알파 버전**을 만든다. 정적 PPT 발표 vs 실제 작동하는 화면 시연은 **심사위원 인상이 5배 차이**.

### 8주 알파 개발 (본선 발표 D-Day 기준 역산)

| 주차 | 산출물 | 본선 발표 활용 |
|:---:|-------|-------------|
| W1 | 인프라 + 인증 (OAuth) | 보안 견고함 어필 |
| W2 | User Service + 결제 통합 | 비즈니스 모델 검증 |
| W3 | 진단 테스트 엔진 (15문항) | **발표 데모 1**: 진단 흐름 |
| W4 | AI 로드맵 생성 (Claude API) | **발표 데모 2**: 로드맵 생성 시연 |
| W5 | AI 멘토 v1 (Claude API + 기본 프롬프트) | **발표 데모 3**: AI 답변 시연 |
| W6 | LCS 브릿지 + Q&A 게시판 | **발표 데모 4 (핵심)**: LCS 동작 |
| W7 | UI 폴리싱 + 베타 모집 페이지 | 사용자 모집 준비 |
| W8 | 베타 100명 초청 + 피드백 수집 | 사용자 데이터 발표 |

### 본선 발표 시점에 보유할 자산

1. **동작하는 알파 버전** (라이브 데모 가능)

3. **사용자 인터뷰 영상 2~3편** (감정적 임팩트)

## 🚨 가장 위험한 함정 5가지 (반드시 회피)

### 함정 1: "통과했으니 한숨 돌리기"

1차 통과는 **시작점**이지 안정 구간이 아닙니다. 합격 일주일 내에 다음 단계 준비를 시작하지 않으면, 3개월 후 본선에서 다른 통과자에게 밀립니다.

**회피**: D+3 이내에 본 문서의 Week 1 체크리스트 시작.

### 함정 2: "AI"만 강조하는 안주

1차 통과자의 30%가 AI 보유. 본선부터는 "AI"는 차별점이 아닙니다.

**회피**: LCS, lived experience, 기존 결과물 3가지 메시지로 재구성.

### 함정 3: 보육기관에 수동적

보육기관은 인센티브 받는 사업 파트너입니다. 적극적으로 요청·활용해야 합니다.

**회피**: 첫 미팅 전 "요청 리스트" 작성, 매주 진척 공유 정기화.

### 함정 4: MVP 개발 vs 발표 준비의 선택 오류

둘 다 해야 합니다. 한쪽만 집중하면 다른 쪽에서 무너집니다.

**회피**: 주 40시간 기준, 25시간 MVP + 15시간 발표·미팅 권장.

### 함정 5: 본선 통과를 너무 강하게 가정

모두의 창업은 단계마다 70~80%가 탈락하는 구조일 수 있습니다.

**회피**: 시나리오 B(예창패 백업) 항상 준비. 1차 통과로 자만하지 말 것.
