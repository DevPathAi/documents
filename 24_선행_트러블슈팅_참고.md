## 2. DevPath AI 직결 교훈 → 적용

### 이미 우리가 겪은 것 (검증된 교훈)
- **gradlew 권한 (#4)**: Windows에서 만든 `gradlew`가 Linux CI에서 `Permission denied`. → `git update-index --chmod=+x gradlew`로 해결함. **신규 백엔드 레포 생성 시 즉시 적용**.
- **gh-pages 서브경로 (#7)**: Vite `base` / VitePress `base`를 `/레포명/`으로. workflow-dashboard·guide에 적용됨.
- **node24 / lockfile 크로스플랫폼**: CI Node 버전과 옵셔널 의존성(@emnapi 등) 불일치. → CI `npm install` + Node 24로 해결함.

### W1 인프라에 직접 반영할 것
- **Flyway core vs starter 침묵 실패 (#11)** + **전역 버전 중복 (#15)**: 중앙집중 스키마(devpath-shared)에서 마이그레이션 버전이 한 시퀀스이므로 **버전 충돌 위험이 더 크다**. → 버전 네이밍 규약(타임스탬프 기반 `V20260615120000__`)과 CI에서 `flyway validate` 가드.
- **앱 ↔ 인프라 계약 불일치 (#2) / env 주입 ≠ 앱 배선 (#13) / relaxed-binding 오배선 (#10)**: docker-compose·gitops가 환경변수를 주입해도 `application.yml`이 그 키를 안 읽으면 침묵 실패. → PG 전환 시 **드라이버·URL·dialect 키를 실제로 읽는지 통합 테스트로 검증**.
- **문서 드리프트 / 픽션 (#9, #12)**: MySQL→PostgreSQL 전환 시 02_ERD·03_아키텍처를 **같은 PR에서 갱신**(코드와 문서 동시). 검증 안 된 내용을 문서에 쓰지 않는다.
- **Boot4 메이저 연쇄 (#3)**: Jackson 3 · ArchUnit 등 Boot4 호환 버전을 BOM으로 고정.
- **이미지 태그 비결정성 (#1, #23)**: gitops `image: ...:latest` 대신 **불변 태그(커밋 SHA)** 권장.

### Kafka 도입 시 (W2+ Outbox, W12 리뷰 워커)
- **토픽 환경 프리픽스 격리 (06-12 #199)**: dev/staging이 공유 Kafka·컨슈머그룹을 쓰면 이벤트가 환경 간 누수된다(dev 검색 0건, staging DLQ). → **토픽·컨슈머그룹에 환경 프리픽스**.
- **Avro/스키마 정본 분기 → DLT**: writer 스키마가 갈리면 역직렬화 실패. → 스키마 레지스트리 + 호환성 테스트.

## 3. 6대 교훈 수렴 (전체 작업 원칙)

Synapse가 78건에서 수렴한 원칙 — DevPath의 [절대 작업 원칙](https://github.com/DevPathAi)과 일치한다.

1. **멱등·단조 불변식을 쓰기 시점에 강제** — 식별자·버전·이벤트는 생성이 아닌 결정적 규칙으로.
2. **단일 출처 단방향 동기화** — 데이터·문서는 한 출처에서 한 방향으로(대시보드 sync, 문서 SSOT).
3. **가정이 아닌 환경 신호로 검증** — "될 것"이 아니라 실제 로그·응답·상태로 확인 (= 추측 금지).
4. **잘못된 머지는 Revert 후 재작성** — 강제 수정 대신 되돌리고 다시.
5. **상태는 origin 기준** — 로컬 stale 오판 방지, 원격을 진실로.
6. **표준은 CI 가드로** — 규약을 사람 기억이 아닌 자동 검사로 강제.
