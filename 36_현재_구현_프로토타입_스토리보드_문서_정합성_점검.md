> **작성일**: 2026-06-19

## 1. 종합 판정

현재 산출물은 큰 방향에서는 정합하다. 제품의 핵심 메시지는 **DevPath AI = 진단 -> 경로 -> 실습 실패 -> LCS 기반 리뷰 -> 다음 학습 보정**으로 수렴했다.

다만 문서 일부는 아직 "목표 아키텍처", "사업계획서용 표현", "현재 구현"을 한 문장 안에 섞어 쓴다. 이 때문에 심사/멘토 공유 시 다음 오해가 생길 수 있다.

| 구분 | 현재 상태 | 판정 |
|---|---|---|
| 핵심 내러티브 | 프로토타입·스토리보드 모두 LCS 우선 흐름으로 정렬 | 정합 |
| 현재 구현 | 인증/사용자, 진단, 게스트 claim, ai-svc Ollama gateway 중심 | 부분 구현 |
| 프론트 | web/admin 목 API 골든패스, mobile 기본 앱 수준 | 문서와 대체로 정합 |
| 백엔드 | community/sandbox/LCS 실제 컨트롤러 없음 | 문서에서 "구현됨"처럼 읽히면 불일치 |
| 문서 | 17번 스케줄이 구현 현황을 가장 잘 반영 | 04/06/07/20/29는 목표 스펙 성격 명시 필요 |

## 2. 현재 구현 기준선

### 2.1 실제 백엔드 컨트롤러

현재 구현된 HTTP 표면은 다음 범위다.

| 서비스 | 구현된 컨트롤러/엔드포인트 |
|---|---|
| `devpath-platform-svc` | `POST /auth/refresh`, `POST /auth/logout`, `GET /users/me` |
| `devpath-learning-svc` | `POST /onboarding/assessments`, `GET /onboarding/assessments/{id}/next`, `POST /answer`, `POST /complete`, `GET /result`, `POST /onboarding/assessments/guest`, guest `next/answer/complete`, `POST /onboarding/assessments/claim` |
| `devpath-ai-svc` | `POST /ai/embed`, `POST /ai/path/generate` |
| `devpath-community-svc` | 컨트롤러 없음 |
| `devpath-sandbox-svc` | 컨트롤러 없음 |

### 2.2 실제 데이터/이벤트 기준

- `devpath-shared`에는 `users`, `user_oauth_identities`, `user_profiles`, `outbox`, `notifications`, `question_bank`, `assessments`, `assessment_items`, `assessment_results` 마이그레이션이 있다.
- 공유 이벤트 record는 `UserRegisteredEvent`, `AssessmentCompletedEvent`, `LearningPathGeneratedEvent`가 있다.
- 현재 코드 기준 `ai-svc`는 Claude 직접 클라이언트가 아니라 **Ollama gateway**다. `/ai/embed`는 768차원 임베딩을 검증하고, `/ai/path/generate`는 구조화 경로 JSON을 검증한다.

### 2.3 실제 프론트 기준

- `devpath-frontend/apps/web`에는 `/login`, `/diagnostic`, `/onboarding`, `/dashboard`, `/path`, `/content/:id`, `/sandbox`, `/mentor`, `/community`, `/community/:id`, `/auth/callback` 라우트가 있다.
- `apps/admin`은 `/dashboard`, `/users`, `/reports` 중심이다.
- `apps/mobile`은 아직 제품 골든패스가 아니라 Flutter 기본 앱 수준이다.

## 5. 핵심 불일치

### D1. "현재 구현"과 "전체 목표 스펙" 경계가 흐림

04/06/07/20은 전체 제품 목표를 다루고, 현재 구현은 MD1 일부다. 문서 독자가 "API 명세에 있으니 구현됨"으로 읽으면 오해한다.

**권장**: 04 상단에 "본 문서는 v1 목표 계약이며 현재 구현 상태는 17/36 기준" 문구 추가.

### D2. AI provider 표현이 분기됨

- 구현/프로토타입: `Ollama`, `nomic-embed-text`, 768차원.
- 사업/아키텍처 목표: `Claude`, `Sonnet`, `Haiku`, `text-embedding-3-small`.

이는 전략적으로 문제는 아니지만, **현재 개발 provider**와 **운영 목표 provider**를 구분하지 않으면 불일치처럼 보인다.

**권장**: 문서 전반에서 "AI Gateway 계약은 provider-agnostic, 현재 개발 빌드는 Ollama, 운영 목표는 Claude/OpenAI 계열"로 통일.

### D3. ERD 임베딩 차원

02는 `VECTOR(1536)`으로 남아 있었고, 현재 슬라이스 #3 설계/구현은 `VECTOR(768)`이다.

**조치**: 본 점검에서 02를 768 기준으로 갱신.

### D4. MVP 범위 문서 간 충돌

27은 8주 알파에서 Sandbox Runner를 "의도적으로 제외"한다고 쓰지만, 최신 17은 MD2에서 콘텐츠 -> Sandbox -> AI 코드리뷰를 2nd Aha 필수 데모로 둔다.

**권장**: 27을 "2026-04-23 멘토링 초안"으로 유지하되, 상단에 "최신 실행 일정은 17 기준" 배너 추가.

### D5. LCS의 구현 상태

프로토타입과 스토리보드는 LCS를 핵심 경험으로 보여주지만, 실제 LCS API/DB/consumer는 아직 없다.

**판정**: 내러티브와 설계는 정합. 구현 완료로 말하면 불일치.

## 7. 현재 한 문장 기준

외부 공유 시 현재 상태는 이렇게 말하는 것이 가장 정합하다.

> DevPath AI는 현재 OAuth/진단/게스트 이관/AI Gateway 기초 구현과 Flutter Web 목 API 골든패스를 갖추고 있으며, 공개 프로토타입과 스토리보드는 LCS 우선 3분 데모로 최신 제품 내러티브를 검증한다. 실제 LCS·커뮤니티·Sandbox 운영 구현은 17번 스케줄의 MD2~MD3 범위다.
