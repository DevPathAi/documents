# GLM-5.2 — AI provider 후보 검토 (Ollama 관점)

> 2026-06-23. 슬라이스 #6 AI 코드리뷰의 provider 전략(Claude=prod / Ollama qwen2.5-coder=dev / Mock=CI) 맥락에서 GLM-5.2(Z.ai/Zhipu)를 평가. **결정이 아니라 후보 기록**이며, 채택 전 골든 eval 실측이 전제다.

## 검증된 사실 (2026-06 출시, 웹 실측)

- **Ollama 라이브러리 등록됨**: `ollama.com/library/glm-5.2`(≈47K pulls). GLM-5.1·glm-5·glm-4.7·glm-4.7-flash(30B)·glm4(9B)도 존재.
- **744B MoE**(활성 ≈40B/토큰), MIT 라이선스·오픈웨이트, 코딩/장기작업 특화, (Z.ai) 1M 컨텍스트.
- **Ollama 사용 경로 2가지**:
  1. **`glm-5.2:cloud`**: 로컬 추론이 아님 — Z.ai 클라우드 API로 라우팅(인터넷+Ollama 계정).
  2. **로컬 GGUF**: 공식 Ollama 로컬 양자화 태그 부재. Unsloth GGUF+llama.cpp 직접 구동. **2-bit도 241GB** → 256GB 통합메모리 Mac 또는 256~300GB RAM+다중 GPU 필요.
- 비용(Cloudflare Workers AI 표기): 입력 $1.40/M, 출력 $4.40/M. 컨텍스트는 플랫폼별 상이(Z.ai 1M vs Cloudflare 262K = 양자화/플랫폼 차이).

## 이 프로젝트 적합성

현 dev 머신은 7B를 CPU ~35s/리뷰 수준(골든 eval 환경).

| 용도 | 판정 | 근거 |
|---|---|---|
| 로컬 dev/CI provider | ❌ 부적합 | 744B/241GB+ — 로컬 구동 불가. dev 티어는 작은 로컬 모델이 맞음 |
| 운영 provider 대안 | ✅ 검토 가치 | Claude(Sonnet $3/$15) 대비 저렴($1.40/$4.40), 오픈웨이트=데이터주권(프로젝트 "한국 리전·데이터주권" 정합), 코딩 특화, 1M 컨텍스트 |

## 통합 난이도

`AiReviewClient` 추상화 + `OllamaAiReviewClient`(`/api/chat`+`format` JSON schema) 이미 존재 →
- GLM-5.2가 Ollama 호환 엔드포인트(`:cloud`/셀프호스트)로 `/api/chat`+구조화출력 지원 시 **`REVIEW_OLLAMA_MODEL` 환경변수 교체만으로 드롭인**(코드 무변경).
- Z.ai OpenAI 호환 API 사용 시 `GlmAiReviewClient` 1개 추가(C2 패턴) 또는 별 provider.

## 권고

1. **로컬 dev 모델 교체 비권장**(하드웨어 불가). GLM 계열 로컬 희망 시 현실 후보는 `glm-4.7-flash`(30B, 여전히 무거움) 또는 `glm4:9b`(qwen2.5-coder:7b 체급).
2. **운영 provider 대안으로 평가** — Claude vs GLM-5.2 비용·데이터주권·코딩품질 trade-off. 경로: Z.ai API/`:cloud`(관리형) 또는 셀프호스트(vLLM, 256GB+; MD4 FinOps/보안 단계 연계).
3. **채택 전 골든 eval 실측**(`documents/docs/superpowers/reports/2026-06-23-slice6-golden-eval.md` 하니스 재사용) — 한국어 구조화출력 품질·검출율을 qwen2.5-coder·Claude와 비교.
