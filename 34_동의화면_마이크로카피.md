# DevPath AI 사용자 동의 화면 마이크로카피

> **문서 목적**: 개인정보 처리방침의 법적 내용을 사용자가 실제로 마주하는 각 화면에서 **이해 가능하고 법적 요건을 충족하는 언어**로 번역한 문서.
>
> **적용 원칙**:
> - 법적 필수 요건 준수 (개인정보 보호법 + 2025.4. 작성지침)
> - 이해도 우선: 전문 용어 최소화, 예시로 설명
> - 투명성: "무엇을, 왜, 얼마나, 어떻게" 네 가지를 모두 포함
> - 선택권 보장: 거부·철회가 항상 가능함을 명시
> - 개발자 타겟 톤: 과도한 친근함 배제, 정확하고 간결
>
> **법적 면책**: 본 초안은 실제 서비스 배포 전 개인정보 전문 변호사 검토 필수.
>
> **문서 버전**: v1.0 (2026-04-22)

## 1. 톤 앤 보이스 기준

### 1-1. 해야 할 것

- **구체적 숫자**: "잠시 후" → "7일 후"
- **이유 설명**: "동의해 주세요" → "AI 답변 품질을 높이기 위해"
- **예시 제공**: "민감정보" → "이메일, API 키, 비밀번호 등"
- **결과 예고**: "거부하시겠어요?" → "거부하셔도 학습 기능은 정상 이용 가능합니다"

### 1-2. 피해야 할 것

- "귀하" 과도 사용 (한 번 사용 후 "이용자" 또는 생략)
- 법조문 인용 (제XX조) — 본문은 처리방침에, 화면은 풀어서 설명
- 이모지 남발 (기능 라벨 외 금지)
- 공포 유발 표현 ("반드시", "의무적으로")

### 1-3. 법적 요건 필수 포함 요소

각 동의 화면은 「개인정보 보호법」 제15조 2항에 따라 다음을 포함해야 합니다.

1. 개인정보의 수집·이용 목적
2. 수집하려는 개인정보의 항목
3. 개인정보의 보유 및 이용 기간
4. 동의를 거부할 권리가 있다는 사실 및 동의 거부에 따른 불이익 내용

이 네 가지가 모든 동의 화면에 반드시 있어야 합니다.

## 2. 회원가입 동의 화면

### 2-1. 전체 약관 화면 구성

```
┌─────────────────────────────────────────┐
│ [← 뒤로]       약관 동의                │
├─────────────────────────────────────────┤
│                                         │
│  시작하기 전에                          │
│  몇 가지 동의가 필요합니다              │
│                                         │
│  ────────────────────────────────       │
│                                         │
│  ☐ 전체 동의                            │
│     ─────────                           │
│                                         │
│  [필수] 만 14세 이상입니다              │
│                              [보기 ›]   │
│                                         │
│  [필수] 서비스 이용약관                 │
│                              [보기 ›]   │
│                                         │
│  [필수] 개인정보 수집·이용 동의         │
│                              [보기 ›]   │
│                                         │
│  [필수] 국외 이전 동의                  │
│                              [보기 ›]   │
│                                         │
│  [선택] 마케팅 정보 수신                │
│                              [보기 ›]   │
│                                         │
│  [선택] 프로모션 이벤트 참여            │
│                              [보기 ›]   │
│                                         │
├─────────────────────────────────────────┤
│                                         │

│                          필수 3개 동의 │
│                                         │
└─────────────────────────────────────────┘
```

### 2-2. 헤드라인

**A안 (권장)**
> **시작하기 전에**
> 몇 가지 동의가 필요합니다

**B안 (간결)**
> **약관 동의**
> 필수 항목은 서비스 이용에 반드시 필요합니다

### 2-3. 각 항목 타이틀 문구

| 구분 | 화면 표시 | 상세 내용 요약 |
|------|----------|--------------|
| 필수 | 만 14세 이상입니다 | 만 14세 미만은 가입할 수 없어요 |
| 필수 | 서비스 이용약관 | 서비스 이용 규칙 |
| 필수 | 개인정보 수집·이용 동의 | 서비스 제공에 필요한 최소한의 정보 |
| 필수 | 국외 이전 동의 (Claude API 이용) | AI 답변 생성을 위한 국외 전송 |
| 선택 | 마케팅 정보 수신 | 이벤트·업데이트 소식 (언제든 해지 가능) |
| 선택 | 프로모션 이벤트 참여 | 할인·이벤트 자동 참여 |

### 2-4. "필수 개인정보 수집·이용 동의" 상세 내용

**[보기] 클릭 시 모달**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
개인정보 수집·이용 동의 (필수)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

DevPath AI 서비스 제공을 위해 다음 정보를
수집·이용하고자 합니다.

▸ 수집 항목
  이메일, 비밀번호(이메일 가입 시)
  또는 OAuth 식별자(소셜 로그인 시)
  닉네임

▸ 이용 목적
  회원 식별 및 인증
  학습 서비스 제공
  공지사항 전달

▸ 보유 기간
  회원 탈퇴 시까지
  (법령에 따라 일부 정보는 분리 보관)

▸ 거부할 권리
  동의를 거부할 수 있으나, 필수 항목을
  거부할 경우 서비스를 이용할 수 없습니다.

자세한 내용은 [개인정보 처리방침]에서
확인하실 수 있습니다.

                              [확인]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 2-5. CTA 버튼 상태

| 상태 | 버튼 텍스트 | 활성화 조건 |
|------|-----------|-----------|
| 비활성 | 필수 항목에 동의해주세요 | 필수 항목 3개 중 하나라도 미체크 |
| 활성 | 동의하고 시작하기 | 필수 항목 3개 모두 체크 |

### 2-6. 작은 설명 텍스트 (화면 하단)

> 필수 항목은 서비스 이용을 위해 반드시 필요해요. 선택 항목은 언제든 설정에서 변경할 수 있어요.

## 3. 선택 동의 화면

### 3-1. 마케팅 정보 수신 (선택)

**[보기] 클릭 시 모달**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
마케팅 정보 수신 동의 (선택)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

▸ 수집·이용 항목
  이메일 주소, 닉네임

▸ 이용 목적
  이벤트 소식 안내
  신규 콘텐츠 업데이트 알림
  맞춤 학습 추천

▸ 발송 방법
  이메일, 푸시 알림

▸ 보유 기간
  동의 철회 시 또는 회원 탈퇴 시까지

▸ 동의 거부 시
  마케팅 정보를 받지 못할 뿐,
  서비스 이용에는 지장이 없어요.

▸ 거부 또는 철회 방법
  언제든 설정 > 알림에서 변경 가능
  수신한 이메일 하단 "수신거부" 링크

                   [동의 안 함]  [동의]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 3-2. 개별 동의 원칙

**중요**: 2024년 9월 개인정보 보호법 개정에 따라 **필수동의 관행이 개선**되었습니다. 선택 항목을 사실상 강제하는 설계(체크 안 하면 회원가입 못 하는 식)는 금지됩니다.

**잘못된 설계 (금지)**
```
☑ 전체 동의 (필수+선택 포함)
→ 사용자가 선택을 의식하지 못하게 유도
```

**올바른 설계 (권장)**
```
☐ 전체 동의
  ─
  ● 필수 항목 (체크박스 개별)
  ○ 선택 항목 (체크박스 개별, 기본 OFF)
```

### 3-3. 선택 항목 기본값 원칙

| 항목 | 기본값 | 이유 |
|------|:-----:|------|
| 마케팅 수신 | OFF | 명시적 opt-in |
| 프로모션 참여 | OFF | 명시적 opt-in |
| GitHub 연동 | OFF | OAuth 별도 플로우 |
| 학습 맥락 수집 | — | LCS 최초 사용 시 별도 화면 |

### 4-2. 권한 스코프 명시

**실제 요청하는 GitHub OAuth 스코프**: `read:user`, `public_repo`

이를 사용자에게 풀어서 설명:

> **"공개 저장소와 프로필 정보를 읽을 수 있어요. 쓰기 권한은 요청하지 않습니다."**

### 4-3. 거부 시 안내

**사용자가 "나중에" 선택 시**

> GitHub 없이도 학습 경로는 잘 만들어져요.
> 진단 테스트 결과를 기반으로 추천됩니다.

## 5. 학습 맥락 자동 첨부 동의 (LCS)

이 화면이 가장 중요합니다. LCS의 차별화 기능이면서 동시에 프라이버시 민감도가 높아 투명성과 이해도가 결정적입니다.

### 5-1. LCS 최초 사용 시 팝업

**사용자가 처음 "질문하기" 버튼을 누를 때** 등장.

```
┌─────────────────────────────────────────┐
│                                         │
│  🎯                                     │
│                                         │
│  답변 품질을 높이기 위해                │
│  학습 맥락을 첨부할 수 있어요           │
│                                         │
│  ─────────────────────                  │
│                                         │
│  DevPath AI는 답변자가 여러분의         │
│  상황을 이해할 수 있도록, 질문 작성 시  │
│  다음 정보를 자동 첨부할 수 있습니다.   │
│                                         │
│  기본 활성화 (언제든 끌 수 있어요)      │
│  ✓ 현재 학습 중인 콘텐츠                │
│  ✓ 학습 경로와 진도                     │
│  ✓ 활성 태그 (Spring, JPA 등)           │
│  ✓ 학습 레벨과 태그별 평판              │
│                                         │
│  기본 비활성화 (민감할 수 있어서)       │
│  ○ 실습 중 발생한 에러 로그             │
│     → 개인정보 자동 마스킹 적용         │
│     → [자세히]                          │
│                                         │
│  ─────────────────────                  │
│                                         │
│  모두 답변자에게만 표시되며,            │
│  질문 작성 시 수정·제외할 수 있어요.    │
│                                         │
├─────────────────────────────────────────┤
│                                         │
│  [ 설정 보기 ]                          │
│  [ 기본 설정으로 시작 ]  강조           │
│                                         │
└─────────────────────────────────────────┘
```

### 5-2. 핵심 문구 A/B 후보

**"왜 수집하는가"에 대한 설명**

- A안: "답변 품질을 높이기 위해"
- B안: "답변자가 여러분의 상황을 정확히 이해하도록"
- C안: "질문 작성 시간을 줄이고 답변을 정확하게 받기 위해"

**"민감도" 표현**

- A안: "민감할 수 있어서 기본 비활성화" ← 권장
- B안: "선택 기능"
- C안: "개인정보가 포함될 수 있어 기본 OFF"

### 5-3. 질문 작성 화면 내 맥락 카드

매번 질문을 쓸 때마다 보여주는 요약 카드:

```
┌───────────────────────────────────────┐
│ ⚡ 학습 맥락 자동 첨부     [ON ⊙⊙]   │
├───────────────────────────────────────┤
│ 📚 학습 중: JPA N+1 문제 해결 (65%)  │
│ 🎯 경로: 3개월 백엔드 로드맵 (3/12)  │
│ 🏷 태그: jpa · spring-boot           │
├───────────────────────────────────────┤
│ 답변자에게 표시돼요 · [항목 수정]    │
└───────────────────────────────────────┘
```

**토글 off 시 문구**

> 맥락 없이 질문하기
> 답변까지 시간이 더 걸릴 수 있어요

## 6. 에러 로그 수집 명시적 동의

DevPath AI 전체에서 **가장 민감한 동의 화면**. 기본 OFF이며 사용자가 명시적으로 활성화해야 합니다.

### 6-1. 에러 로그 활성화 화면 (설정에서 직접 켜려고 할 때)

```
┌─────────────────────────────────────────┐
│                                         │
│  ⚠️                                     │
│                                         │
│  에러 로그 첨부를 켜시겠어요?           │
│                                         │
│  ─────────────────────                  │
```

## 14. 공통 UI 가이드

### 14-1. 체크박스 규격

| 상태 | 사양 |
|------|------|
| 크기 | 20×20 (최소 터치 타겟 44×44 포함) |
| 간격 | 체크박스와 라벨 사이 12px |
| 라벨 탭 영역 | 라벨 전체가 탭 가능해야 함 |
| 필수 표시 | `[필수]` 접두어 (text-xs, 500) |
| 선택 표시 | `[선택]` 접두어 (text-xs, 400) |

### 14-2. "자세히 보기" 링크

모든 동의 항목에 제공. 클릭 시 처리방침의 해당 섹션으로 딥링크 또는 모달.

### 14-3. 접근성 요구사항

- 모든 체크박스에 `aria-label` 또는 연결된 `<label>`
- 필수 항목은 `aria-required="true"`
- 에러 메시지는 `role="alert"` + 시각 표시
- 키보드만으로 완주 가능 (Tab 순서)

### 14-4. 다크 모드

모든 동의 화면 다크 모드 대응. 특히 경고 문구는 다크에서도 적절한 대비.

### 14-5. 다국어 준비

MVP는 한국어만. 그러나 스트링은 i18n 구조로 분리하여 향후 영어 추가 용이하게.

### 14-6. 모바일 우선 설계

- 모든 모달·팝업이 모바일 375×812 기준 스크롤 가능
- 약관 텍스트는 최대 너비 제한 (readability)
- 긴 문서는 "전체 보기" 별도 페이지로 분리
