# 20. 커뮤니티 기능 설계서

> **문서 목적**: 커뮤니티를 "학습과 동등한 축"으로 편입하는 기능·아키텍처·UX 스펙.
>
> **문서 성격**: v1 목표 커뮤니티 설계. 현재 `devpath-community-svc`는 스켈레톤 상태이며, 본 문서는 구현 완료 보고가 아니라 후속 구현 기준이다. 전체 현황은 [37_전체_문서_전체_레포_정합성_점검](./37_전체_문서_전체_레포_정합성_점검.md)을 기준으로 확인한다.
>
> **핵심 전제**
> - 커뮤니티 개방도: **회원 한정**
> - AI ↔ 커뮤니티: **AI 1차 → 커뮤니티 에스컬레이션**
> - 게이미피케이션: **스택오버플로 모델** (평판·배지·레벨)
> - 모더레이션: **장기 자정 시스템** (신뢰 권한 위임)
> - 포지셔닝: **학습과 동등한 차별화 축**
>
> **차별화 포지션**: "학습 맥락을 아는 유일한 한국 개발자 커뮤니티"

## 2. 게시판 타입 및 기능 스펙

### 2.1 다섯 가지 게시판 구성

#### ① Q&A 게시판 (핵심 · 스택오버플로 모델)

**용도**: 기술 질문, 명확한 답이 있는 문제 해결형.

**글 구조**
- 제목 (필수, 80자 제한)
- 본문 (Markdown + 코드 블록 + 이미지)
- **학습 맥락 자동 첨부** (차별화 — §5 상술)
- 태그 (1~5개, 자동완성)
- 질문 품질 체크리스트 (선택)
  - [ ] 시도한 방법 포함
  - [ ] 에러 메시지 포함
  - [ ] 재현 가능한 코드 포함

**답변 구조**
- Markdown + 코드
- 답변 당 **하나만 채택 가능** (질문자 권한)
- 투표 (upvote/downvote) — Phase 2부터

**평판 보상**
| 액션 | 점수 |
|------|------|
| 내 답변 upvote | +10 |
| 내 질문 upvote | +5 |
| 내 답변 채택됨 | +15 |
| 내 답변이 채택한 사람의 answer | +2 |
| 내 글 downvote | −2 |
| downvote 행사 (비용) | −1 |

#### ② 자유 게시판

**용도**: 커리어·회사·취업·일상. 답이 없는 토론형.

**글 구조**
- 제목 + 본문 (Markdown)
- 카테고리 (커리어/취업/회사생활/잡담/후기/뉴스 공유)
- 태그 (0~3개)

**답변 구조**
- 일반 댓글형 (스레드 없음)
- 좋아요 / 싫어요

**평판 보상**: 일반 댓글은 평판에 영향 없음 (자유 게시판이 평판 파밍 지름길이 되는 것 방지). 단 **인기글(좋아요 50+)은 글쓴이에게 +20 보너스**.

#### ③ 스터디 모집

**용도**: 그룹 스터디·프로젝트 팀 모집.

**구조화 필드**
- 스터디 주제
- 인원 (현재/최대)
- 기간
- 방식 (온라인/오프라인/하이브리드)
- 지역 (오프라인인 경우)
- 요구 레벨 (신입/주니어/시니어/무관)
- 연락 방식

**참여 플로우**
1. "참여 신청" 버튼 → 쪽지 자동 전송 + 대기 리스트 등록
2. 개설자가 승인/거절
3. 승인 시 전용 스터디 채팅방 자동 생성 (Phase 2부터)

**평판 보상**: 없음. 단 "완주한 스터디" 이력이 프로필 뱃지로 기록.

#### ④ 프로젝트 공유

**용도**: 개인 프로젝트·포트폴리오·사이드 프로젝트 쇼케이스.

**구조화 필드**
- GitHub 링크 (필수)
- 배포 URL (옵션)
- 사용 스택 (태그)
- 프로젝트 타입 (토이/사이드/창업)
- 피드백 요청 여부
- 기여자 모집 여부

**피드백 메커니즘**
- 일반 댓글
- **"코드 리뷰" 버튼** — Q&A 게시판 질문 자동 생성 + 원글 링크
- **"스타" 기능** (좋아요 강화판)

**평판 보상**
- 스타 1개: +3
- 스타 50개 이상: "주목받는 프로젝트" 뱃지 + 대시보드 노출

#### ⑤ 수료자 라운지 (Phase 2)

**용도**: 학습 경로 완주자 전용. 심화 토론·이직 정보·오프라인 모임.

**접근 권한**: 학습 경로 1개 이상 완주.

**평판 보상**: 기본값 × 1.5배 (전문성 인센티브).

### 2.2 게시판 공통 기능

**알림**
- 내 글에 댓글/답변
- 내가 답변한 질문이 채택됨
- 내가 팔로우한 사용자의 새 글
- 내 프로필에 팔로워 추가

**검색** (Elasticsearch 기반, 기존 콘텐츠 검색과 통합)
- 필터: 게시판 / 태그 / 기간 / 답변 있음 / 해결됨 / 작성자
- 정렬: 최신순 / 인기순 / 미답변 / 답변 많은 순

**팔로우·북마크**
- 사용자 팔로우
- 태그 팔로우 (새 글 알림)
- 글 북마크 (폴더 관리)

**신고**
- 카테고리: 스팸/욕설/광고/중복/부적절/기타
- AI 자동 모더레이션 1차 → 신뢰 사용자 리뷰 2차 → 관리자 최종

## 3. 스택오버플로 모델 평판 시스템

### 3.1 평판 점수 (Reputation)

**획득 규칙**
| 액션 | 점수 | 비고 |
|------|------|------|
| 질문 upvote | +5 | |
| 답변 upvote | +10 | |
| 답변 채택됨 | +15 | |
| 내 답변 질문자에게 채택됐을 때 | +2 | 추가 보너스 |
| 자유게시판 인기글 달성 | +20 | 좋아요 50+ |
| 프로젝트 스타 받음 | +3 (각) | |
| 편집 승인 (타인 글 개선) | +2 | |
| 현상금 지불 받음 | 질문자가 건 금액 | 평판 소모 |

**차감 규칙**
| 액션 | 점수 |
|------|------|
| 내 글 downvote 받음 | −2 |
| downvote 행사 | −1 (비용) |
| 답변 삭제됨 (모더레이터) | −10 |
| 스팸 판정 | −100 |

### 3.2 레벨 및 권한 언락

> **MVP 기준**: 커뮤니티 글쓰기(Q&A, 자유게시판, 프로젝트 공유) 횟수 제한 없음. 유료 플랜 도입 시 제한 적용 예정 (21_유료_플랜_참고설계 참고).

| 평판 | 권한 |
|------|------|
| 1 (가입) | 글쓰기, 댓글, 채택 (MVP: 무제한) |
| 15 | 질문에 upvote |
| 50 | 답변에 댓글 |
| 125 | 답변에 downvote (비용 −1) |
| 250 | 태그 위키 편집 제안 |
| 500 | 타인 글 편집 제안 |
| 1,000 | 투표 신뢰 부여, **답변에 현상금 걸기** |
| 2,000 | 편집 직접 승인 |
| 3,000 | 중복 질문 플래그, 닫기 투표 |
| 5,000 | 태그 위키 직접 편집 |
| 10,000 | **신뢰 사용자** (사용자 삭제 투표, 모더레이션 큐 접근) |
| 20,000 | 답변 보호 (신규 답변 제한) |

### 3.3 배지 시스템

**Bronze (흔함)**
| 배지명 | 조건 |
|--------|------|
| 첫 걸음 | 프로필 작성 완료 |
| 첫 질문 | 질문 작성 |
| 첫 답변 | 답변 작성 |
| 학생 | 질문 1개가 +1 이상 |
| 선생 | 답변 1개가 +1 이상 |
| 자선가 | 평판 15 도달 |
| 비평가 | downvote 1회 행사 |
| 편집자 | 글 편집 1회 |
| 커뮤니티 | 30일 연속 방문 |

**Silver (의미 있는 활동)**
| 배지명 | 조건 |
|--------|------|
| 훌륭한 답변 | 답변이 +25 |
| 인기 질문 | 질문 조회수 1,000+ |
| 멘토 | 답변 10개 채택 |
| 계몽가 | 처음으로 채택된 답변이 +10 이상 |
| 분류자 | 5개 태그에서 각 20+ 답변 |
| 신뢰받음 | 평판 1,000 도달 |
| 비평가 견습 | 채택 답변 100개 보유 |

**Gold (희귀, 상징적)**
| 배지명 | 조건 |
|--------|------|
| 전설적 답변 | 답변이 +100 |
| 명성 | 질문 조회수 10,000+ |
| 학자 | 평판 10,000 도달 |
| 태그 마스터 | 단일 태그에서 1,000+ 평판 |
| 베테랑 | 가입 2년 경과 + 평판 3,000 |
| 스승 | 답변 50개 채택 |
| 신뢰 사용자 | 신뢰 사용자 권한 획득 |

### 3.4 태그별 평판

**개념**: 총 평판 외에 태그별로도 평판 점수를 별도 집계.

- 예: 홍길동 · 총 평판 3,450 / Spring 1,200 / JPA 820 / Kubernetes 15
- "Kubernetes 질문에는 낯선 사람" 같은 맥락 파악 가능
- 답변자 신뢰도 계산 시 **태그별 평판이 더 중요** (총점보다)

### 3.5 평판 남용 방지

- 같은 사용자로부터 하루 +40 점수 제한 (부정 투표 방지)
- 새 계정은 첫 7일 upvote/downvote 불가
- IP·디바이스 기반 sockpuppet 탐지
- 평판 조작 의심 시 자동 감사 (Moderation Service)

## 4. AI ↔ 커뮤니티 에스컬레이션 플로우

### 4.1 기본 플로우

```text
사용자 질문 입력
       │
       ▼
┌──────────────────┐
│  AI 1차 답변     │  ← Claude API + RAG (학습 맥락 + 유사 질문 DB)
│  (평균 3~5초)    │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 사용자 평가      │
│ [도움됨][불충분] │
└──┬────────────┬──┘
   │            │
   ▼ 도움됨     ▼ 불충분
 종료         ┌──────────────────┐
 (+AI 답변이   │ 커뮤니티 에스컬   │
  검색DB에     │ 레이션 옵션 제시 │
  기록)        └────────┬─────────┘
                        │
          ┌─────────────┼─────────────┐
          ▼             ▼             ▼
    커뮤니티에       현상금 걸기    수료자 라운지
    공개 질문        (평판 차감)    (심화 태그)
          │             │             │
          └─────┬───────┴─────────────┘
                ▼
         답변 수집
                │
                ▼
         채택 → 평판 부여 → 검색 DB 인덱싱
```

### 4.2 에스컬레이션 옵션 UX

**AI 답변 하단 노출**
```
┌────────────────────────────────────────────┐
│ AI 답변이 도움이 되셨나요?                │
│                                            │
│ [✓ 네, 해결됐어요]                        │
│ [✗ 부족해요, 커뮤니티에 물어볼래요]       │
└────────────────────────────────────────────┘
```

**"부족해요" 선택 시**
```
┌────────────────────────────────────────────┐
│ 커뮤니티에 질문하기                        │
│                                            │
│ AI 답변과 함께 제공하면 답변이 빨라져요.   │
│                                            │
│ ☑ AI 답변 첨부 (권장)                     │
│ ☑ 현재 학습 맥락 첨부 (권장)              │
│ ☑ 에러 로그 첨부                          │
│ ☐ 현상금 걸기 (+50 평판 소모)             │
│                                            │
│ [질문 작성으로]                            │
└────────────────────────────────────────────┘
```

### 4.3 AI 답변 재활용

- 새 질문 작성 시 기존 AI 답변과 유사도 검사 (pgvector)
- 유사도 **80% 이상**이면 "비슷한 질문에 AI가 답한 적이 있습니다" 안내
- 이 메커니즘이 **중복 질문을 자연스럽게 감소**

### 4.4 현상금(Bounty) 시스템

**조건**
- 평판 **1,000 이상** 사용자만 가능
- 최소 50 ~ 최대 500 평판 포인트
- 걸린 평판은 **즉시 차감**
- 7일 내 답변 없으면 최고 투표 답변에 절반 자동 지급
- 그마저 없으면 평판 소멸

## 5. "학습 맥락 연결" 차별화 메커니즘

> 다른 한국 커뮤니티와의 **가장 강력한 차별점**.

### 5.1 질문 작성 시 자동 첨부 맥락

사용자가 "질문하기" 버튼을 누른 시점 상태가 자동으로 메타데이터로 첨부됩니다.

**자동 수집 정보**
```json
{
  "learning_context": {
    "current_content": {
      "id": "content-jpa-nplus1",
      "title": "JPA N+1 문제 해결",
      "step": "1/5",
      "progress": 0.65
    },
    "learning_path": {
      "id": "path-backend-3month",
      "title": "3개월 백엔드 로드맵",
      "completed_steps": 3,
      "total_steps": 12
    },
    "user_level": {
      "track": "backend",
      "diagnosed_level": "junior",
      "tag_reputation": {
        "jpa": 120,
        "spring": 240
      }
    },
    "recent_errors": [
      {
        "timestamp": "2026-04-22T10:30:00Z",
        "source": "sandbox",
        "message": "LazyInitializationException: failed to lazily initialize..."
      }
    ]
  }
}
```

### 5.2 답변자 UI에 표시되는 맥락

```
┌──────────────────────────────────────────────────┐
│ 질문자 맥락 (DevPath AI가 자동 수집)            │
├──────────────────────────────────────────────────┤
│ 📚 학습 중: JPA N+1 문제 해결 (Step 1/5 · 65%)  │
│ 🎯 경로: 3개월 백엔드 로드맵 (3/12 단계 완료)   │
│ 👤 레벨: 주니어 / JPA 평판 120                  │
│ ⚠️  최근 에러: LazyInitializationException      │
│                                                  │
│ [상세 보기]                                      │
└──────────────────────────────────────────────────┘
```

### 5.3 왜 이게 강력한가

**답변자 관점**
- "이 사람이 어느 수준인지"를 알고 답변 깊이 조절 가능
- "어떤 콘텐츠에서 막혔는지"를 알고 맥락 맞는 답변 가능
- "방금 어떤 에러를 봤는지"를 알고 정확히 진단 가능

**질문자 관점**
- 질문 작성 시간 절약 (배경 설명 불필요)
- 질문 품질 자동 향상 → 답변률 증가
- 스스로도 "내가 지금 뭘 하고 있는지" 구조화 가능

**플랫폼 관점**
- 맥락 데이터로 자동 태깅 가능
- 유사 질문 매칭 정확도 급상승
- "이 콘텐츠에서 자주 막히는 지점"을 학습 팀이 파악 → **콘텐츠 개선 루프**

### 5.4 프라이버시

- 자동 수집은 **명시적 opt-in 토글** (항목별 기본값 상이 — PIA §5-2 기준)
  - 현재 학습 경로/진척률: 기본 **ON**
  - `recent_errors` (에러 로그): 기본 **OFF** (민감정보 포함 가능, 23_PIA §5-2 준수)
- 각 정보 항목 개별 on/off 가능
- 첨부 전 사용자가 미리보기 가능
- 공개 범위 선택: "답변자만" / "공개"

## 6. 콜드 스타트 문제 해결

> 회원 한정 커뮤니티의 최대 리스크: 초기 "질문 있는데 답변자 없음" 상태. 3단계 중첩 해결.

### 6.1 AI 시드 답변 (Day 1부터)

**메커니즘**
- 모든 Q&A 질문에 자동으로 **AI 답변이 먼저 달림** (Claude API)
- AI 답변은 "🤖 AI 초안 · 인간 답변이 달리면 보완됩니다" 뱃지
- AI 답변도 채택 가능하되 평판은 안 줌

**효과**: 답변 대기 시간 0. 질문자가 "답이 없다"는 경험을 절대 하지 않음.

### 6.2 초기 시드 사용자 프로그램

**"Founding Contributors" 50명 모집** (런칭 전 3개월)

- 조건: 특정 분야 경력 3년+ / 블로그나 오픈소스 활동
- 혜택:
  - 프로필에 "Founding Contributor" 영구 배지
  - 초기 평판 500 부여 (권한 일부 해금 상태)
  - 답변 1개당 **5,000원** 소액 페이 (초기 3개월, 월 상한 30만원/인, 총 예산 450만원)
  - Pro 플랜 6개월 무료 (수익화 시점에 적용)
- 모집 채널 및 타임라인:

| 시점 | 채널 | 목표 |
|------|------|------|
| Week 5-8 | 본인 네트워크 직접 접촉 | 10명 (핵심 멤버, 1:1 DM) |
| Week 9-12 | 한국 개발자 슬랙/디스코드 (OKKY, 코딩각, 스프링 한국 등) | +15명 |
| Week 13-16 | Twitter/X + 개발 블로그 기고 | +15명 |
| Week 17-20 | 기존 멤버의 추천 (레퍼럴 보너스 평판 +100) | +10명 |

- 목표 KPI: 런칭 1개월차에 질문당 평균 답변 **1.5개**

### 6.2.1 Community Guidelines 초안

> DevPath AI 커뮤니티는 **학습자가 안전하게 질문하고, 경험 있는 개발자가 기꺼이 답하는 공간**입니다.

**핵심 원칙:**
1. **정중하게** — 질문에 바보 같은 질문은 없습니다. 답변에 무시나 비꼼은 없습니다.
2. **구체적으로** — 코드, 에러 메시지, 시도한 것을 포함해 주세요. 학습 맥락 첨부를 활용하세요.
3. **건설적으로** — "이건 틀렸어"보다 "이렇게 하면 더 나아요 + 이유"가 좋습니다.
4. **원본을 존중** — 타인의 코드나 답변을 무단 복사하지 마세요. 출처를 밝히세요.

**금지 사항:**
- 스팸, 광고, 자기 홍보 (프로젝트 공유 게시판은 허용)
- 혐오 발언, 차별, 괴롭힘
- 의도적 오답, 악의적 코드 공유
- AI 생성 답변을 직접 작성한 것처럼 게시 (AI 시드 답변은 시스템이 명시)

**제재 단계:** 경고 → 7일 쓰기 정지 → 30일 쓰기 정지 → 영구 정지. 이의제기는 24~72시간 내 처리.

### 6.3 "지금 답하면 좋은 질문" 큐

**미답변 질문 가시성 극대화**
- 홈 대시보드에 "답변 기다리는 질문 3개" 카드 상시 노출
- 내가 강한 태그 영역에 미답변 질문 생기면 푸시 알림
- 답변 없이 24시간 지난 질문은 자동 승격 (보상 +50% 임시)

### 6.4 운영 초기 전략 (첫 6개월)

| 개월 | 전략 |
|------|------|
| M1 | Founding Contributors 활동 + AI 시드 답변 + 본인 직접 답변 |
| M2-3 | 유저 유입 가속, "좋은 질문" 큐레이션해 트위터 공유 |
| M4-6 | 평판 상위 유저에게 권한 이양, 자정 시스템 가동 |

## 7. 모더레이션 및 자정 시스템

### 7.1 3단계 방어선

```
┌─ Layer 1: AI 자동 모더레이션 ─┐
│ 스팸/욕설/광고/NSFW 자동 탐지 │
│ 실시간, 99% 케이스 커버       │
└──────────────┬────────────────┘
               ▼
┌─ Layer 2: 신뢰 사용자 검토 ──┐
│ 평판 10,000+ 유저의 투표    │
│ 3표 일치 시 자동 처리       │
└──────────────┬────────────────┘
               ▼
┌─ Layer 3: 관리자 최종 판단 ──┐
│ 분쟁 케이스, 제재 결정       │
│ 본인 + 향후 전담 인력        │
└───────────────────────────────┘
```

### 7.2 AI 자동 모더레이션

**사용 모델**: Claude Haiku + 한국어 욕설/스팸 규칙 기반 사전 필터

**검출 항목**
- 욕설·혐오 표현
- 스팸성 홍보 (학원·부트캠프 광고 등)
- 명백한 오답 (기본 사실 오류)
- 중복 질문
- 규정 위반 (이력서 공유, 개인정보 노출)

**대응**
| 심각도 | 자동 대응 |
|--------|----------|
| Low | 경고 플래그 (사용자에게 표시) |
| Medium | 게시 보류 + 작성자 알림 |
| High | 즉시 비공개 + 관리자 큐 |
| Critical | 즉시 삭제 + 계정 일시 정지 |

### 7.3 신뢰 사용자 권한 (평판 10,000+)

**언락 기능**
- 모더레이션 큐 접근
- 신고된 글 리뷰 투표
- 중복 질문 닫기 투표
- 태그 위키 편집

**"3표 규칙"**: 신뢰 사용자 3명 동일 판단 시 자동 처리. 2:1 분쟁 시 관리자 큐.

### 7.4 사용자 제재 체계

| 제재 | 조건 | 효과 |
|------|------|------|
| 경고 | 첫 위반 | 팝업 안내, 평판 −10 |
| 쓰기 일시 정지 (7일) | 2회 위반 또는 심각 1회 | 읽기만 가능 |
| 쓰기 정지 (30일) | 반복 위반 | 읽기만 가능 |
| 계정 정지 | 심각한 위반 | 로그인 불가 |
| 영구 밴 | 악의적 반복 | 이메일·디바이스 블랙리스트 |

### 7.5 이의제기

- 모든 제재에 대해 이의제기 가능
- 관리자 24~72시간 내 응답
- 과도한 제재 확인 시 **평판 복구 + 기록 삭제**

## 10. 차별화 3대 리스크 대응 전략 (CEO 리뷰 보강 7)

### 리스크 1: "학습 맥락 자동 첨부"가 실제로 답변 품질을 높이는지 검증 방법 부재

**대응: 검증 프로토타입**
- 개발 Week 5-8에 간이 프로토타입으로 A/B 테스트
- 맥락 첨부 ON vs OFF 시 답변자의 "답변하고 싶다" 의향 비교
- Founding Contributors 초기 멤버 10명으로 내부 테스트

**검증 지표:**

| 지표 | 기준 | 측정 시점 |
|------|------|----------|
| 맥락 첨부 질문의 답변률 vs 미첨부 | 첨부가 1.5배 이상 | M3 이후 |
| 맥락 첨부 질문의 답변 시간 vs 미첨부 | 첨부가 30% 빠름 | M3 이후 |
| 답변자 "맥락이 도움되었다" 비율 | 70% 이상 | M3 이후 |

### 리스크 2: AI 시드 답변이 너무 좋으면 인간 답변 동기 소멸

**대응: AI 시드 품질 의도적 제한 + 점진적 노출**
1. AI 시드는 "방향 제시" 수준으로 제한 (완전한 해결책 X)
2. AI 시드는 **접혀있는 상태로 기본 표시** (CEO 리뷰 결정)
   - "AI 초안 보기" 클릭으로 펼침
   - 인간 답변 영역이 시각적으로 더 강조됨 (상단 + accent 색상)
3. 인간 답변이 달리면 AI 시드를 자동 축소 (하단으로 이동)
4. **측정 및 임계값:**
   - 경고 임계값: AI 시드만으로 해결 70% 초과 시 프롬프트 리뷰
   - 행동 임계값: 80% 초과 시 프롬프트 즉시 조정
   - 목표 범위: 40~60%
5. **운영 대시보드** (CEO 리뷰 Section 8 결정):
   - AI 시드 해결률 실시간 그래프 + 70%/80% 임계값 알림
   - 데이터 플라이휠(확장 #3) Admin 대시보드와 통합

### 리스크 3: 다운보트 문화가 한국에 맞는지

**대응: 단계적 도입 + 한국화 조정**
1. Phase 1 (MVP): **다운보트 비활성화**. upvote + 채택만 운영
2. Phase 2 (M7~): 평판 125+ 사용자에게만 다운보트 활성화
   - 다운보트 시 본인 평판 -1 차감
   - 다운보트 사유 필수 입력 (익명 다운보트 방지)
3. **측정**: 다운보트 받은 사용자의 7일 내 재방문율 70% 미만이면 제도 재검토

## 11. Kafka 이벤트 계약 (Eng 리뷰 보강)

### 이벤트 토픽 구조

| 이벤트 | 토픽 | 파티션 키 | Consumer Group | 재시도 정책 |
|--------|------|-----------|---------------|------------|
| CommunityQuestionPostedEvent | `community-questions` | `question_id` | `ai-seed-worker`, `ai-mod-worker`, `ai-quality-worker` | 3회, exponential backoff |
| LearningContextSnapshotRequestedEvent | `community-context` | `user_id` | `lcs-worker` | 3회, exponential backoff |
| CommunityAiAnswerEscalatedEvent | `community-escalation` | `question_id` | `escalation-worker` | 3회, exponential backoff |
| CommunityQuestionUnanswered24hEvent | `community-escalation` | `question_id` | `escalation-worker` | 1회 (cron 재발행) |
| CommunityQuestionUnanswered72hEvent | `community-escalation` | `question_id` | `ai-seed-worker` | 3회, exponential backoff |
| CommunityUserTrustLevelChangedEvent | `community-reputation` | `user_id` | `badge-worker`, `notification-worker` | 3회, exponential backoff |

### AI 병렬 처리 패턴 (CEO 리뷰 Section 1 결정)

질문 작성 시 `CommunityQuestionPostedEvent`를 **fan-out** 패턴으로 3개 consumer group이 동시 소비:
- `ai-seed-worker`: AI 시드 답변 생성 (Claude Haiku, ~2-3초)
- `ai-mod-worker`: AI 모더레이션 심각도 판정 (Claude Haiku, ~1-2초)
- `ai-quality-worker`: 질문 품질 AI 스코어링 (Claude Haiku, ~1-2초)

**체감 시간:** 병렬 처리로 3-5초 (직렬 시 8-15초)

역링크 콘텐츠 추천은 답변 표시 후 비동기로 처리 (사용자 대기 없음).

### Graceful Degradation (CEO 리뷰 Section 2 결정)

| AI 호출 | 실패 시 fallback | 사용자 경험 |
|---------|-----------------|-----------|
| AI 시드 답변 | 질문만 게시, "AI 답변 준비 중" 표시 후 비동기 재시도 | 질문 게시는 블로킹 안 됨 |
| AI 모더레이션 | 규칙 기반 필터(keyword/regex) fallback. 규칙 통과면 게시, 규칙도 실패면 보류 | 게시 지연 없음 (Eng 리뷰 절충안) |
| 질문 품질 스코어링 | 점수 없이 게시 | 개선 제안 미표시 |
| 역링크 콘텐츠 추천 | 추천 영역 미표시 | 기능 없는 것처럼 보임 |

### 보안: 프롬프트 인젝션 이중 방어 (CEO 리뷰 Section 3 결정)

1. **입력 살균**: 사용자 질문 텍스트에서 시스템 프롬프트 조작 패턴 필터링
2. **출력 검증**: AI 생성 답변에 모더레이션 레이어 적용 (악성 콘텐츠/링크/코드 검출)
3. 시스템 프롬프트에 인젝션 방어 지시 포함

## 12. CEO 리뷰 확장 기능 (Cherry-pick 수용)

### 확장 1: 질문 품질 AI 스코어링

질문 작성 시 AI가 자동으로 품질을 1-5점으로 점수화하고, 낮은 점수(3점 이하)에는 개선 제안을 표시합니다.

**성공 지표:** 스코어 3점 이하 질문의 개선 후 답변률 +20%

### 확장 2: 커뮤니티 → 학습 콘텐츠 역링크

Q&A 답변에서 언급된 개념과 관련된 학습 콘텐츠를 pgvector 기반으로 자동 추천합니다. "동등한 축" 양방향 루프 완성.

**기술:** pgvector (PostgreSQL 15+), 임베딩 모델 multilingual-e5-base 등 (미선정, Week 4 전 확정)
**성공 지표:** 역링크 클릭률 10%+, 학습 재진입률 15%+

### 확장 3: 콘텐츠 병목 데이터 플라이휠

학습 맥락 스냅샷을 집계하여 "이 콘텐츠의 어디에서 사람들이 막히는가"를 Admin 대시보드에 표시합니다.

**구현:** 배치 집계 (일 1회 cron), Admin 대시보드 패널
**성공 지표:** 병목 데이터로 콘텐츠 수정 후 해당 지점 질문 30% 감소
