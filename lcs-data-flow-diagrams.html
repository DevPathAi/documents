LCS · Data Flow Diagrams
Learning Context Snapshot System · DevPath AI · 2026-04-22
D1

Write Path · Event Collection

학습 서비스들이 이벤트를 발행 → Kafka → LCS Consumer가 Redis 캐시에 누적 갱신

⚠ 민감 경로
Sandbox error 이벤트만 Sanitize 단계를 거침. 다른 이벤트는 메타데이터만 있으므로 우회.
Idempotency
event_id를 Redis Set에 1시간 TTL로 기록 → 중복 처리 방지. LWW(Last Write Wins) + timestamp로 out-of-order 이벤트 무시.
장애 격리
LCS 전체 다운 → 학습 서비스 영향 0. Kafka가 이벤트 쌓고 있다가 복구 시 catch-up. Eventual consistency 허용.
D2

Read Path · Draft to Commit Sequence

유저가 질문 작성 시작 → 미리보기 → 질문 게시까지 시퀀스

✓ Fast Path
Cache hit인 경우 P95 40ms. 유저가 "질문하기" 누르고 맥락 카드 보기까지 체감 즉시.
Fallback
캐시 miss (초기 접속, staleness 10분+) 시 3개 서비스 병렬 호출. P95 120ms. Partial success 허용.
Dual Lifecycle
Draft는 Redis TTL 10분 (휘발). Commit 시에만 PostgreSQL에 영속 저장. 게시 안 된 draft는 자동 소멸.
D3

Sanitize Pipeline · 3-Stage Defense

원본 에러 로그에서 민감정보를 제거하는 파이프라인 — 시스템의 프라이버시 최전선

⚠ Fail-Safe 원칙
Sanitize 실패 시 에러 첨부 포기. "첨부 실패 > 민ађinga 유출". 다른 맥락 필드는 유지.
Hot-Reloadable Rules
새 민감 패턴 발견 시 DB에 추가하면 즉시 적용. 코드 배포 없음. Stage 3가 탐지한 패턴 → Stage 2 룰로 승격.
유저 원본 접근
유저는 자신의 원본 에러를 디버깅 목적으로 볼 수 있음 (Sandbox Session에 암호화 저장). Snapshot에는 Sanitized만 들어감.
