# 19. 온보딩 와이어프레임 스펙

> **문서 목적**: Figma에서 바로 와이어프레임을 구축할 수 있는 수준의 픽셀 단위 스펙. 디자인 토큰, 컴포넌트 라이브러리, 화면별 구성, 상태, 인터랙션 포함.
> **적용 범위**: Step 0 (랜딩) ~ Step 5 (Sandbox 실습 + AI 코드 리뷰 2nd Aha) 온보딩 플로우
>
> **뷰포트**
> - 모바일 primary: **375 × 812** (iPhone 13 mini 기준)
> - 데스크톱: **1440 × 900** (MacBook Air 기준)
> - 태블릿: 모바일 레이아웃을 최대 너비 600px로 스케일
>
> **단위**: 모든 크기는 px. **8-point grid** 준수 (8의 배수 권장, 4px은 예외적으로 허용)

### 1.3 스페이싱

8-point grid. 공백은 이 값 중에서만 선택.

| 토큰 | 값 | 주요 용도 |
|------|-----|----------|
| `space/0` | 0 | — |
| `space/1` | 4 | 아이콘-텍스트 사이 |
| `space/2` | 8 | 인라인 요소 간 |
| `space/3` | 12 | 작은 gap |
| `space/4` | 16 | 기본 gap, 카드 내부 |
| `space/5` | 20 | 중간 gap |
| `space/6` | 24 | 섹션 내부 gap |
| `space/8` | 32 | 섹션 간 gap |
| `space/10` | 40 | 큰 섹션 분리 |
| `space/12` | 48 | 페이지 상단/하단 여백 |
| `space/16` | 64 | 데스크톱 대형 여백 |

**화면 좌우 여백 (horizontal padding)**
- 모바일: `space/5` (20)
- 태블릿: `space/8` (32)
- 데스크톱: `space/16` (64), 단 max-width 1200 초과 시 auto 센터링

### 1.4 라운드
| 토큰 | 값 | 용도 |
|------|-----|------|
| `radius/xs` | 4 | 뱃지, 작은 태그 |
| `radius/sm` | 8 | 입력 필드, 작은 버튼 |
| `radius/md` | 12 | 기본 카드, CTA 버튼 |
| `radius/lg` | 16 | 큰 카드, 모달 |
| `radius/xl` | 24 | 히어로 이미지, 특수 카드 |
| `radius/full` | 9999 | 원형 아바타, 칩 |

### 1.5 그림자
| 토큰 | 값 | 용도 |
|------|-----|------|
| `shadow/xs` | `0 1px 2px rgba(15,23,42,0.06)` | 입력 필드 포커스 |
| `shadow/sm` | `0 2px 4px rgba(15,23,42,0.08)` | 작은 카드 |
| `shadow/md` | `0 4px 12px rgba(15,23,42,0.10)` | 기본 카드 hover |
| `shadow/lg` | `0 10px 24px rgba(15,23,42,0.14)` | 모달, 드롭다운 |
| `shadow/xl` | `0 20px 48px rgba(15,23,42,0.18)` | 히어로 플로팅 요소 |
| `shadow/inner` | `inset 0 1px 2px rgba(15,23,42,0.06)` | 입력 필드 내부 |

### 1.6 애니메이션 토큰

## 2. 컴포넌트 라이브러리

### 2.1 Button

**공통 스펙**
- Height: Primary 56 / Secondary 48 / Small 40
- Padding horizontal: 24 / 20 / 16
- Border radius: `radius/md` (12)
- Typography: `text/heading-sm` (16/24, 600)
- Icon slot (옵션): 20×20, 텍스트와 gap 8

**Variants**
| Variant | 배경 | 텍스트 | 보더 | 용도 |
|---------|------|--------|------|------|
| Primary | `color/primary/500` | `color/neutral/0` | 없음 | 주요 CTA |
| Secondary | `color/neutral/0` | `color/primary/700` | 1.5px `color/neutral/300` | 보조 액션 |
| Ghost | 투명 | `color/primary/500` | 없음 | 3순위 액션 |
| Destructive | `color/neutral/0` | `color/error/500` | 1.5px `color/error/500` | 위험 액션 |

**States**
| State | 변화 |
|-------|------|
| Default | 기본 토큰 적용 |
| Hover | 배경 +8% darker, `shadow/sm` 추가 |
| Active (press) | `scale(0.98)`, `shadow/xs` |
| Focus | outline 2px `color/primary/300`, offset 2px |
| Disabled | opacity 0.4, cursor not-allowed |
| Loading | 텍스트 숨김, 20×20 스피너 표시 |

**Full-width 규칙**: 모바일 하단 고정 CTA는 100% width, horizontal padding `space/5`.

### 2.2 Input Field

**스펙**
- Height: 56
- Padding: horizontal 16, vertical 16
- Border: 1.5px `color/neutral/300`, radius `radius/sm` (8)
- Typography: `text/body-md` (15/24)
- Placeholder: `color/neutral/500`

**Label 배치**
- Label 위 배치, gap 8
- Typography: `text/body-sm` (13/20, 500)
- Color: `color/neutral/700`

**Helper text / Error**
- Input 아래, gap 6
- Typography: `text/caption` (12/18)
- Helper: `color/neutral/500`
- Error: `color/error/500`, 좌측 ⚠ 아이콘 14×14 gap 4

**States**
| State | 변화 |
|-------|------|
| Default | 기본 |
| Focus | border 2px `color/primary/500`, `shadow/xs` |
| Filled | border 1.5px `color/neutral/700` |
| Error | border 2px `color/error/500` |
| Disabled | background `color/neutral/100`, text opacity 0.5 |

**스펙**

**토큰**

- 바 채워짐: `color/primary/500` / 트랙: `color/neutral/200`, radius full

### 2.6 Loading Stepper (경로 생성 중)

**스펙**
- 각 스텝 row: height 32, vertical gap 12
- 아이콘: 20×20 (체크 / 스피너 / 대기 점 3개)
- 텍스트: `text/body-md`

**아이콘 상태**
- 완료: ✓ `color/success/500`
- 진행 중: 스피너 (1.2s rotate) `color/primary/500`
- 대기: ○ outline `color/neutral/300`

### 2.7 Toast / Snackbar

**스펙**
- 위치: 모바일 하단 center, bottom 16px (safe-area 고려)
- Max-width: 335 (모바일) / 400 (데스크톱)
- Padding: 12 16
- Background: `color/neutral/950`
- Text: `color/neutral/0`, `text/body-sm` 500
- Radius: `radius/md` (12)
- Shadow: `shadow/lg`
- 자동 제거: 4초 (에러는 6초)

## 4. 상태 스펙

### 4.1 공통 상태 처리
모든 화면은 다음 5가지 상태를 고려합니다.

| 상태 | 처리 |
|------|------|
| Default | 정상 렌더링 |
| Loading | 스켈레톤 또는 스피너 (데이터 > 500ms 시) |
| Empty | 일러스트 + 안내 + 추천 액션 CTA |
| Error | 인라인 메시지 또는 toast (심각도에 따라) |
| Offline | 상단 배너 "오프라인 모드 — 읽기만 가능" |

### 4.2 Skeleton 컴포넌트
- Background: `color/neutral/100`
- Shimmer: linear-gradient animation 1.5s infinite
- Radius: 원본 컴포넌트의 radius 토큰 동일 적용
- Height: 원본 높이 동일

**사용 위치**
- 경로 생성 로딩 화면 하단 (선택)
- 콘텐츠 로드 중

### 4.3 에러 상태 분류
| 에러 유형 | 표현 | 예시 |
|----------|------|------|
| Validation | 입력 필드 하단 inline | 이메일 형식 오류 |
| Network | Toast (bottom, 6초) | 일시적 연결 끊김 |
| Critical | 모달 (차단형) | 인증 실패 |
| Server (5xx) | 풀스크린 empty state | API 장애 |

## 6. 접근성 스펙

### 6.1 대비
- 본문 텍스트 대비 최소 **4.5:1** (WCAG AA)
- 큰 텍스트 (18pt+) 최소 3:1
- CTA 버튼 대비 최소 4.5:1 (배경 vs 텍스트)

### 6.2 터치 타겟
- 최소 **44×44** (iOS HIG) 또는 **48×48dp** (Android)
- 인접 터치 타겟 간 최소 간격 8px

### 6.3 포커스 표시
- 모든 interactive 요소에 focus ring 표시
- Ring: 2px outline `color/primary/300`, offset 2px
- 키보드 내비게이션 순서 명확히 (tab-index 관리)

### 6.4 스크린 리더
- 모든 버튼에 `aria-label` 제공 (아이콘 전용 버튼 필수)
- 진행률: `role="progressbar"` + `aria-valuenow` / `aria-valuemax`
- 코드 블록: `<pre>` + `<code>` + `lang` attribute
- 이미지: `alt` 텍스트 또는 `aria-hidden="true"` (장식용)

### 6.5 모션 감소
- `prefers-reduced-motion: reduce` 미디어쿼리 적용
- 화면 전환 200ms 이하로 단축 또는 즉시 전환
- 자동 재생 영상 (랜딩 히어로) 정지

### 7.2 Variants 사용 권장
- **Button**: variant (Primary/Secondary/Ghost/Destructive) × state (Default/Hover/Active/Disabled/Loading) × size (Large/Medium/Small)
- **Option Card**: state (Default/Hover/Selected/Disabled) × layout (Single-line/Two-line)
- **Progress Bar**: type (Dots/Percent) × totalSteps (4/15)

### 7.3 Auto Layout 설정
모든 컴포넌트 Auto Layout 필수. Direction·gap은 토큰 값 사용 (`space/3=12`, `space/4=16` 등).

### 7.4 이름 규칙
- 컴포넌트: `Button/Primary/Default/Large`
- 토큰: `color/primary/500`, `space/4`, `radius/md`
- 프레임: `O03-Quiz-Q03-Default`
- 레이어: kebab-case 한글 허용 `step-list`, `option-card-1`
