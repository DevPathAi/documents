# P. 문제인식 (Problem)

## 1-1. 창업 아이템의 개발 동기

### 1-1-1. 개인적 동기 (창업자 경험)

현재 개발 실무 경력 12년차인 본 창업자는, 2년제 대학 인터넷정보처리학과에서 개발을 전공하고 학원을 수료한 후 개발자로 첫발을 뗐습니다. 그러나 실무에 들어가자 학교와 학원에서 배운 내용만으로는 턱없이 부족했습니다.

가장 답답했던 것은 "부족한 부분을 어디서 어떻게 채워야 하는지" 알 수 없었다는 점입니다. 학원은 이미 수료한 상태였고, 온라인 강의는 본인 수준에 맞춰주지 않았으며, 회사에는 가르쳐주는 사수가 없었습니다. "알아서 찾아봐"가 전부였습니다.

결국 혼자 삽질하며 Spring Boot, React, Flutter, Docker, Kafka까지 독학으로 익혔습니다. **"지금 네 수준에서는 이걸 이 순서로 공부해"라고 한마디만 해줬어도 최소 6개월은 아꼈을 것**입니다.

독학하며 주변을 관찰한 결과, 이는 개인의 문제가 아니라 구조적 문제임을 확인했습니다. 부트캠프를 수료하고도 취업에 고전하는 지인, 비전공 독학자로 스타트업에 들어갔지만 번아웃으로 퇴사한 사례, 현직 주니어이면서도 "내 기술이 제대로 된 것인지" 불안해하는 동료. 공통점은 모두 **"지금 내가 어디에 있고, 다음에 뭘 해야 하는지"를 알려주는 사람이 없다**는 점이었습니다.

그로부터 12년, 본 창업자는 그때 가장 절실했던 "AI 사수"를 직접 설계·구축할 수 있는 기술 깊이를 갖춘 개발자가 되었습니다. 문제를 직접 산 사람이, 이제 그 문제를 풀 역량까지 갖춘 셈입니다.

### 1-1-2. 아이디어 진화 과정

처음에는 이 문제를 "내 경험을 블로그로 공유하면 되겠다"는 접근으로 시작했으나, 학습자 한 명 한 명의 수준과 막힌 지점이 달라 일반화된 글로는 해결이 안 된다는 점을 깨달았습니다.

같은 "JPA를 어떻게 배우지?"라는 질문에도 어떤 학습자는 SQL 기초부터 필요하고, 어떤 학습자는 N+1 성능 이슈만 보완하면 되는 식입니다. 이 지점에서 **AI의 가능성**을 보았습니다. 사람이 1:1로 붙을 수 없는 규모의 개인화 문제를, AI는 학습자 맥락에 맞춰 풀 수 있습니다.

Claude·GPT-4 수준의 대형 언어모델이 학습 맥락을 이해하고 개인화 답변을 생성할 수 있는 시대가 되었기에, **본 창업자가 독학 시절 가장 간절히 원했던 "AI 사수"를 이제 기술로 만들 수 있다**는 확신에 이르렀습니다.

## 1-2. 목표 고객 및 시장 분석

### 1-2-1. 목표 고객 (페르소나)

**페르소나 1: CS 전공 대학생 (1차 타겟)**

- 상황: 3~4학년, 취업 준비 시점, 이론은 배웠지만 실무 감각 부족
- 페인 포인트: "포트폴리오를 어떤 순서로 만들어야 취업에 유리한지" 가이드 부재
- 지불 의사: 월 14,900원은 학원(월 수십만원) 대비 감당 가능한 수준

**페르소나 2: 부트캠프 수료 직후 신입 (2차 타겟)**

- 규모: K-디지털 트레이닝(KDT) 및 민간 부트캠프 수료생 (연간 규모는 자체 정량조사로 검증 예정, 2026.7~8)
- 상황: 수개월·수백만원 과정을 마쳤으나 수료 후 취업 연계가 끊기는 공백 발생. **국비 정보통신(IT) 직업훈련 평균 취업률은 55.7%**(전국 IT 과정 541개 유효분 자체 집계, HRD-Net 공시 기준)로 전체 직종 평균(53.5%)을 겨우 2%p 웃도는 수준 — '유망 분야' 통념과 달리 수료가 곧 취업을 보장하지 않음
- 페인 포인트: 수료 후 지원 단절, "다음에 뭘 해야 할지" 막막함
- 지불 의사: 이미 수백만원을 쓴 후라 추가 소액 지출 부담 있으나, 취업 실패 시 재투자 필요 (지불의사 정량 검증 예정)

**페르소나 3: 중소·스타트업 현직 주니어 (3차 타겟 / B2C 확장)**

- 규모: 국내 SW개발자 약 **24.8만명**(2024, 과기정통부 SW산업실태조사·SPRi, 통계청 승인통계 제127005호 / 2020년 15.1만명 → 4년간 약 1.6배 증가). 이 중 경력 3년 미만 주니어층이 핵심 타깃 (정확한 주니어 비중은 자체 검증 예정)
- 상황: 회사에 사수 없이 혼자 성장해야 함, AI·클라우드·MSA 트렌드 추격 필요
- 페인 포인트: 책·인강으로 해결 안 되는 "실무 판단력" 부족
- 지불 의사: 커리어 투자 관점에서 월 1.4만원 지출 저항 낮음

> **검증 계획**: 위 페르소나별 지불 의사·수요 강도는 '모두의 창업' AI 솔루션을 활용해 2026.7~8월 정량 검증 예정 (깔로 패널 설문 200+ 응답, AskMind 실사용자 대화 100+ 건). 검증 결과로 본 절의 수치를 갱신함.

### 1-2-2. 시장 규모 (TAM-SAM-SOM)

**TAM (Total Addressable Market) - 전체 시장**

- 한국 에듀테크 시장 전체: 약 **10조 원** (2026년 기준 추정)
- 이 중 **성인 IT 교육 시장**: 약 3,000억 원
- 연 성장률: **12% 이상** (KOCCA·한국에듀테크협회 자료)

**SAM (Serviceable Addressable Market) - 서비스 가능 시장**

- 국내 개발 학습자 연 25만 명
- 평균 지출 가능액 월 2만 원 기준
- 25만 명 × 2만 원 × 12개월 = **연 600억 원**

**SOM (Serviceable Obtainable Market) - 달성 가능 시장**

- 3년 차 점유 목표 5% = **연 30억 원**
- 초기 1년차 점유 1% = **연 4.5억 원** (= 2,500명 × 14,900원 × 12개월)

> **검증 계획**: SAM의 지불 가능액(월 2만 원) 가정은 깔로 정량 설문으로 실측·보정 예정 (2026.7~8).

# S. 실현가능성 (Solution)

## 2-1. 창업 아이템의 구체적 설명

### 2-1-1. 핵심 시스템 설계: 자동 순환 학습 시스템

DevPath AI는 **진단→계획→실행→평가→보정**이 자동 순환하는 5단계 시스템입니다.

**1단계 진단**

- 가입 시 **15문항 적응형 코딩 테스트**로 영역별 실력 측정
- 맞히면 난이도 상향, 틀리면 기초 영역 탐색
- GitHub 연동 시 공개 저장소의 주요 언어·커밋 패턴·프로젝트 규모 추가 분석

**2단계 계획**

- AI가 진단 결과를 기반으로 맞춤 로드맵 생성
- 예: "Java 기초는 되지만 JPA와 네트워크가 약하니 이 순서로 4주간 공부해"
- **이전 단계 통과 → 다음 단계 해제** 구조로 빈틈 없는 성장

**3단계 실행**

- 매일 "오늘 할 것" 자동 제시
- Docker 컨테이너 기반 실습 환경(Sandbox) 즉시 제공
- 학습자가 고민 없이 바로 시작 가능

**4단계 평가**

- 답변 정확도, 학습 시간, 실습 결과 실시간 추적
- 모든 데이터는 Kafka 이벤트 스트림으로 수집 → AI 분석 파이프라인 입력

**5단계 보정**

- 오답률 상승 또는 학습 시간 감소 감지 시 AI가 난이도 조정·복습 삽입·로드맵 수정
- **"인강은 안 보면 끝이지만, DevPath AI는 못 따라가면 자동으로 경로가 바뀐다"**

### 2-1-2. 핵심 차별화 기술: 학습 맥락 자동 첨부 시스템(LCS)

DevPath AI의 **기술적 방어선(moat)**은 "Learning Context Snapshot Service(LCS)"입니다.

**작동 원리**

- 학습자가 AI 멘토에게 질문하거나 커뮤니티에 글을 올릴 때, 현재 학습 중인 콘텐츠·진도·최근 실습 에러 맥락이 **자동으로 첨부**
- 답변자(AI 또는 사람)는 질문자 수준을 알고 **맥락에 맞는 답변** 제공
- "처음부터 설명해 주세요" 없이 즉시 정확한 답변 가능

**기술 아키텍처**

- Kafka 이벤트 스트림으로 학습 활동 실시간 수집
- Redis에 최신 맥락 스냅샷 유지 (TTL 7일)
- 질문 작성 시점에 Post 서비스와 연결되는 **브릿지 아키텍처** (폴리레포 MSA 구조)
- 에러 로그 민감정보(API 키, 이메일, 인증 토큰 등)는 **3단계 Sanitize 파이프라인**으로 자동 마스킹

**경쟁 우위**

- 한국의 어떤 플랫폼(인프런, OKKY, 스택오버플로, 부트캠프)에도 **없는 기능**
- 단순 복제가 어려운 **구조적 차별화** (학습 시스템과 커뮤니티의 데이터 브릿지)
- 학습 데이터가 쌓일수록 정확도 상승 → **데이터 네트워크 효과**
