## 1. 개요

슬라이스 #4 신규 기능 — 콘텐츠 뷰어 조회 + 읽기 진척(scroll/dwell) + 진척 임계 도달 시 ACTIVE 학습경로 task 자동완료 + 대시보드 진척 반영 — 의 끝단간 동작을 로컬에서 검증한다. 슬라이스 #1~#3 E2E(39번 보고서) 패턴을 따른다.

## 2. 검증 환경

- 인프라: PostgreSQL 17(pgvector, 5432)·Redis 7(6379) docker compose healthy. **Kafka·Ollama 미가동** — 본 검증 범위(콘텐츠 조회/진척)는 learning-svc 내부 DB 트랜잭션이라 불요(task 자동완료도 동일 tx의 JdbcTemplate UPDATE).
- 서비스: devpath-learning-svc(dev 프로파일, 8082)·devpath-gateway(8080) 로컬 구동. platform·ai-svc 미구동(콘텐츠 뷰어/진척에 불요).
- 인증: learning·gateway 동일 JWT secret(`application.yml`/`GatewaySecurityConfig` 기본값 일치 확인) → HS256 JWT 직접 생성(`sub`=user_id). iss/aud 미검증·서명+만료만(슬라이스 #1 모델).
- 대상 데이터: 기존 DB. user_id `604923600`의 ACTIVE 학습경로 #1 — week1 task 1/2/3 이 각각 PUBLISHED 콘텐츠 content 4/3/2(spring-quiz/practice/read)에 연결, 검증 시작 시 전부 미완료.

## 5. 프론트엔드 web (mock 모드, :5173)

- flutter web(`USE_MOCK=true`) web-server 구동 → headless 브라우저(gstack)로 접속.
- ✅ 앱 빌드·렌더 정상: 진단 진입("실력 진단 15문항")·진단 문항("1/15 · 샘플 진단 문항", 선택지 A/B/잘 모르겠어요) 화면 렌더 확인(스크린샷 증적).
- ✅ 인터랙션 일부: "진단 시작하기"(ElevatedButton) 클릭 → 문항 화면 전환 동작.
- ⚠️ **콘텐츠 뷰어 화면 브라우저 캡처 미달성**: flutter web CanvasKit + semantic 모드에서 진단 선택지(GestureDetector)에 대한 headless 자동화 탭(semantic click·합성 pointer 이벤트)이 flutter GestureArena로 전달되지 않아 진단 완주 불가 → 인증/온보딩 게이트(`gateRedirect`: `AuthAuthenticated && onboardingStatus==done` 필요) 통과 불가. 콘텐츠 뷰어는 인증 영역(`/content/:id`, ShellRoute) 안이라 도달하지 못했다. 이는 flutter web headless 자동화의 제약이며 슬라이스 #4 코드 결함이 아니다.
- 보완: 콘텐츠 뷰어 UI(빌드 E)는 위젯 테스트로 검증됨 — `apps/web/test/features/content/content_page_test.dart`·`content_progress_smoke_test.dart`(CI `analyze-test` 녹색). 기능 계약은 위 §3/§4 백엔드+gateway로 실증.

## 6. 결론

슬라이스 #4 핵심 기능(콘텐츠 조회 · 진척 monotonic upsert · scroll0.8+dwell45 임계 완료 · ACTIVE path task 자동완료 · 대시보드 progressPercent 반영 · 멱등 · 404/400/401 에러 매핑)은 백엔드(learning-svc)와 gateway 경유로 **끝단간 실증**됐다. 프론트 web은 mock 렌더·인터랙션을 확인했고, 콘텐츠 뷰어 화면 자체는 flutter web headless 자동화 한계로 브라우저 픽셀 캡처를 못 했으나 위젯 테스트로 검증된다.

## 7. 후속(선택)

- **콘텐츠 뷰어 브라우저 풀 E2E**: platform + GitHub OAuth 앱 등록(사용자 선행) 후 실 로그인 경로로 진단→경로생성(Ollama)→뷰어 확인. 또는 flutter `integration_test`+`flutter drive`로 게이트를 코드 레벨에서 우회.
- **B2 콘텐츠 dev seeder 부재**: 현재 메인 DB contents는 11개(슬라이스 #3 잔여)뿐, B2의 승인 콘텐츠 150 seed(`content_md2_seed.sql`)는 메인 DB에 미적재(`QuestionBankSeeder`만 dev seeder 존재, content seeder 없음). dev/demo용 ContentSeeder 추가를 검토한다.
- Kafka·Ollama 포함 전체 끝단간(진단→Ollama 경로생성→onboarding DONE 전이)은 별도 회귀로 둔다(슬라이스 #3 39번 보고서가 해당 경로 커버).
