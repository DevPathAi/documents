- 검증일: 2026-06-20

- 범위: 전 서비스 구동 -> 웹 진단 -> AI 경로 생성 -> 온보딩 DONE -> 경로 화면 확인

## 1. 검증 목적

사용자가 웹에서 진단을 완료하면 learning-svc가 ai-svc/Ollama를 통해 학습 경로를 생성하고, 생성 완료 후 사용자의 온보딩 상태가 DONE으로 전환되며, 프론트엔드에서 생성된 학습 경로 화면을 정상 표시하는지 확인한다.

## 6. 검증 중 발견 및 조치한 이슈

### 6.1 ai-svc: 빈 strengths/weaknesses validation 실패

증상:

- 실제 진단 결과에서 `strengthConcepts` 또는 `weaknessConcepts`가 빈 배열일 수 있었다.
- ai-svc DTO가 `@NotEmpty`를 요구해 `/path/generate` 요청이 400으로 실패했다.

조치:

- `PathGenerateRequest`에서 concept list는 null만 금지하고 빈 배열은 허용하도록 변경했다.
- blank concept element는 계속 거부한다.
- 관련 컨트롤러 테스트를 추가/수정했다.

변경 파일:

```text
devpath-ai-svc/src/main/java/ai/devpath/aigw/ollama/dto/PathGenerateRequest.java
devpath-ai-svc/src/test/java/ai/devpath/aigw/ollama/OllamaControllerTest.java
```

검증:

```text
./gradlew.bat test --tests ai.devpath.aigw.ollama.OllamaControllerTest
```

결과: 통과

### 6.2 learning-svc: SSE 30초 timeout

증상:

- 실제 Ollama `qwen2.5:7b` 생성 시간이 30초를 넘었다.
- learning-svc의 `SseEmitter` timeout이 30초로 고정되어 `AsyncRequestTimeoutException`이 발생했다.

조치:

- SSE timeout을 설정값으로 주입할 수 있게 변경했다.
- 기본값은 180초로 늘리고, E2E 검증에서는 240초로 실행했다.

변경 파일:

```text
devpath-learning-svc/src/main/java/ai/devpath/learning/path/LearningPathController.java
```

검증:

```text
./gradlew.bat classes
```

결과: 통과

### 6.3 frontend: DONE 이후 /path 진입 시 무조건 재생성

증상:

- 학습 경로가 이미 생성되어 user onboarding status가 DONE이어도 `/path` 진입 시 프론트가 무조건 `/learning-paths/me/generate` SSE를 다시 호출했다.
- 이로 인해 기존 ACTIVE path가 있는 상태에서 중복 생성/409 흐름이 발생할 수 있었다.

조치:

- `PathController.loadOrStart()`를 추가했다.
- `/path` 진입 시 먼저 `GET /learning-paths/me`로 기존 ACTIVE 경로를 조회한다.
- 기존 경로가 있으면 바로 완료 화면을 표시하고, 없는 경우에만 새 경로 생성을 시작한다.

변경 파일:

```text
devpath-frontend/apps/web/lib/src/features/path/application/path_controller.dart
devpath-frontend/apps/web/lib/src/features/path/presentation/path_page.dart
devpath-frontend/apps/web/test/features/path/path_controller_test.dart
devpath-frontend/apps/web/test/features/path/path_page_test.dart
```

검증:

```text
flutter test apps/web/test/features/path/path_controller_test.dart apps/web/test/features/path/path_page_test.dart --suppress-analytics --reporter expanded
```

결과: 8개 테스트 통과

### 6.4 frontend: auth bootstrap과 path 조회 경합

증상:

- `/path` 화면이 AuthLoading 중 먼저 렌더되면서 path 조회가 access token 없이 시작되었다.
- 이후 `/auth/refresh`는 성공했지만 최초 path 조회가 이미 실패해 화면이 진행 상태에 머물 수 있었다.

조치:

- `PathPage`가 `AuthAuthenticated` 상태를 확인한 뒤에만 `loadOrStart()`를 호출하도록 변경했다.
- 최종 네트워크에서 `POST /auth/refresh` 200 이후 `GET /learning-paths/me` 200이 확인되었다.

## 8. 미실행/선택 항목

빌드 C 동시성 409 테스트는 선택 항목이며, 이번 검증 범위에서는 의도적으로 정식 테스트로 포함하지 않았다.

단, E2E 검증 중 기존 ACTIVE path가 있는 상태에서 `/path`가 재생성을 시도하는 경합성 409 유사 흐름은 실제로 관찰되었고, 프론트가 기존 경로를 먼저 조회하도록 수정하여 재발 가능성을 낮췄다.

## 9. 결론

전 서비스와 인프라를 실제로 구동한 상태에서 웹 진단부터 Ollama 기반 경로 생성, DB 저장, 사용자 DONE 전환, 최종 경로 화면 표시까지 확인했다.

최종 상태 기준으로 E2E 수동검증은 통과했다.
