## 3. 전환 제안 타이밍 3곳

### ① 진단 테스트 결과 직후 (동기 최고조)
- 학습 경로를 막 받았을 때 → "Pro로 무제한 멘토 + Sandbox"

### ② 첫 과제 성공 직후 (성취감)
- Sandbox 제출 후 AI 리뷰 통과 화면 하단

### ③ 무료 한도 도달 (자연스러운 업셀)
- 멘토 20회 소진, Sandbox 월 100회 소진
- **기능 제한을 이유로 제안** (강제적이어도 수용됨)
- 전환율 대비 이탈 리스크: **낮음** (니즈 기반)

## 5. 결제 실패 · 해지 플로우 카피

### 5.1 결제 실패

> **결제가 처리되지 않았어요**
> 보통 이런 이유 중 하나입니다:
> - 카드 한도 초과
> - 해외 결제 차단 (DevPath는 국내 결제이지만 혹시)
> - 카드 유효기간 만료
>
> [다른 카드로 시도]  [계좌이체]  [고객센터]

### 5.2 해지 시도 (Retention 페이지)

> **떠나시기 전에**
>
> 떠나는 이유를 알려주시면 제품 개선에 반영하고, 맞는 제안을 드릴 수 있습니다.
>
> - 가격이 부담 → 학생 할인 50% 제안 가능
> - 콘텐츠가 부족 → 새로 추가된 콘텐츠 안내
> - 기능이 기대와 달라 → 기본 플랜으로 전환
> - 바빠서 못 써요 → 3개월 일시정지 가능
> - 다른 제품으로 갈아탐 → 피드백 부탁드립니다
>
> [해지 진행]  [제안 보기]

> 💡 해지 페이지의 목표는 **"붙잡기"가 아니라 "이유 파악 + 대안 제시"**. 강하게 붙잡으면 해지는 막아도 브랜드가 상합니다.

## 7. 데이터 모델

### 7.1 `user_plans`

|------|------|------|
| user_id | BIGINT PK FK | |

| external_subscription_id | VARCHAR | PG 측 구독 ID |
| status | ENUM | ACTIVE / CANCELED / EXPIRED / PAUSED |

| 컬럼 | 타입 | 설명 |
|------|------|------|
| user_id | BIGINT FK | |
| period_start | DATE | YYYY-MM-01 |
| ar_sessions_used | INT | |
| mentor_questions_used | INT | |
| sandbox_runs_used | INT | |
| community_qna_used | INT | |
| community_free_used | INT | |
| PRIMARY KEY | (user_id, period_start) | |

### 7.3 `billing_events` (Webhook 멱등 저장)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT PK | |
| provider | VARCHAR | |
| external_event_id | VARCHAR UK | PG 이벤트 ID (dedup) |
| event_type | VARCHAR | subscription.created / renewed / failed / canceled |
| payload | JSONB | |
| processed_at | TIMESTAMPTZ | |
| user_id | BIGINT FK | |

### 7.4 이벤트 (Outbox)
- `PlanUpgradedEvent`: 한도 재계산 + 환영 이메일

- `PlanExpiredEvent`: Free로 다운그레이드
- `PlanPaymentFailedEvent`: Notification + 재시도 스케줄

## 8. API 스펙

### 8.1 플랜
| Method | Endpoint | 설명 | 권한 |
|--------|----------|------|------|
| GET | `/plans` | 플랜 카탈로그 | PUBLIC |
| GET | `/plans/me` | 현재 플랜 + 사용량 | LEARNER |
| POST | `/plans/me/checkout` | Pro 결제 세션 생성 | LEARNER |
| POST | `/plans/me/cancel` | 해지 (만료 시 다운그레이드) | LEARNER |
| POST | `/plans/me/pause` | 일시정지 (최대 3개월) | LEARNER |
| POST | `/plans/me/resume` | 일시정지 해제 | LEARNER |
| POST | `/webhooks/billing` | 결제 프로바이더 webhook | INTERNAL (서명 검증) |

### 8.2 응답 예시

**`GET /plans/me`**
```json
{
  "plan": "PRO",
  "status": "ACTIVE",
  "started_at": "2026-03-15T00:00:00Z",
  "expires_at": "2026-04-15T00:00:00Z",
  "usage": {
    "ar_sessions_used": 12,
    "mentor_questions_used": 34,
    "sandbox_runs_used": 78
  }
}
```

**`POST /plans/me/checkout`**
```json
// Response
{
  "checkout_session_url": "https://payments.toss.im/v1/checkout/...",
  "expires_at": "2026-04-22T11:15:00Z"
}
```

### 8.3 결제 Webhook 흐름
```
PG → POST /webhooks/billing (서명 검증)
  → billing_events.external_event_id UNIQUE (멱등)
  → event_type 분기
      subscription.created  → user_plans INSERT/UPDATE plan=PRO
      subscription.renewed  → user_plans.renewed_at 갱신
      subscription.canceled → user_plans.status=CANCELED (expires_at까지는 PRO 유지)
      payment.failed        → PlanPaymentFailedEvent
  → Outbox: PlanUpgradedEvent 등
```

## 11. 유저 스토리 Epic L5 — 자연스러운 업셀

**Story L5.1 — 절정의 순간에 제안**
> 학습자로서 **내가 가장 만족한 순간에 유료 제안**을 받으면 자연스럽다. 왜냐하면 피로할 때 결제 독촉은 짜증나기 때문이다.

**AC**
- [ ] 진단 결과 직후 작은 Pro 배너 (강제 아님, 3초 후 자동 축소)
- [ ] 첫 과제 성공 직후 축하 + "Pro로 더 많은 실습"
- [ ] 한도 도달 시 "Pro로 무제한" + **대안 (내일까지 기다리기)** 명시

**Story L5.2 — 안심하고 시작하기**
> 사용자로서 **카드 등록 후에도 언제든 해지**할 수 있다는 것을 믿고 싶다.

**AC**
- [ ] 페이월에 "14일 무과금 · 해지 2클릭" 명시
- [ ] PCI-DSS 준수 표기 + 카드 저장 안 함 문구
- [ ] 설정 → 구독 → 해지 경로 2클릭 내

**Story L5.3 — 떠날 때도 나쁜 경험 안 만들기**
> 해지하는 사용자로서 **이유를 말하면 대안이 주어진다**는 느낌을 받고 싶다.

**AC**
- [ ] 5가지 해지 이유 선택지
- [ ] 이유별 맞춤 제안 (가격/콘텐츠/기능/시간/경쟁사)
- [ ] 붙잡으려고 강하게 누르지 않기

## 12. KPI · A/B 테스트 우선순위

### 12.1 유료 관련 KPI (참고)

| 영역 | 지표 | 목표 |
|------|------|------|
| 전환 | 무료 → Pro 전환율 (30일) | ≥ 5% |
| 전환 | 14일 체험 → 유료 유지율 | ≥ 40% |
| 리텐션 | Pro 유저 월 이탈률 | < 8% |
| Retention | 해지 사유 수집률 | ≥ 70% |
| FinOps | Pro 유저 MAU당 마진 | 원가의 4배 이상 |

### 12.2 페이월 A/B 테스트

- **페이월 카피**: "가치 강조" vs "사회적 증거"
- **전환 트리거 위치**: 진단 직후 vs 첫 과제 직후 (효율 비교)
- **가격 제시 방식**: 월 19,900원 vs "커피 3잔 값" vs 연간 할인 제시
- **무료 체험 기간**: 7일 vs 14일 vs 30일

## 13. 리스크 및 Retention

| 리스크 | 대응 |
|--------|------|
| 무료 한도 스트레스로 이탈 | 한도 근접 시 사전 알림 + 대안 제시 |
| 결제 실패로 PRO 끊김 | 재시도 3회 + 이메일 + 7일 grace |
| 체험 후 높은 이탈 | 체험 종료 2일 전 "사용한 기능" 요약 이메일 |
| 해지 사유가 가격이면 | 할인 제안 대신 플랜 다운셀 (기본 플랜 신설 검토) |
| 환불 요구 | 정책 명문화 (사용량 기준) |

---

## 14. 평판 기반 락인 효과 (스택오버플로 패턴)

> 유료화의 숨은 자산은 **사용자가 쌓은 평판**.

- 쌓인 평판은 유료 해지 시 **동결**, 재가입 시 복원
- 평판 1,000+ 사용자는 해지율이 일반의 1/5로 추정 (스택오버플로 벤치마크)
- 배지·태그 전문가 지위 → "내가 쌓은 자산"이 있는 플랫폼은 못 떠남
- 이 효과는 **커뮤니티 + 유료화**의 결합 가치가 큰 이유
