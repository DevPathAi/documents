## 1. 비회원 진단 체험 → 가입 → 1st Aha

> **관련 화면**: SCR-W-LAND-001 (랜딩), SCR-W-ONB-002 (비회원 진단), SCR-W-AUTH-001 (로그인), SCR-W-PATH-001 (학습 경로)

```mermaid
sequenceDiagram
    participant U as 방문자
    participant W as Web
    participant API as API Gateway
    participant ONB as Onboarding
    participant AUTH as Auth (OAuth2)
    participant GH as GitHub API
    participant PATH as Path Engine
    participant AI as AI Gateway (Claude)

    U->>W: 랜딩 진입 (데모 영상 노출)
    W-->>U: CTA "무료로 학습 경로 받기"
    U->>W: 클릭 → 비회원 진단 시작
    W->>API: POST /onboarding/assessments/guest
    API->>ONB: 임시 세션 생성 (Redis 30분)
    loop 15문항 적응형
        W->>API: GET /next → 문항 수신
        U->>W: 답안 입력
        W->>API: POST /answer (난이도 조정)
    end
    W->>API: POST /complete → 결과 "직전" 시점
    API-->>W: "결과를 받으려면 로그인해 주세요"

    U->>W: "GitHub로 계속하기"
    W->>AUTH: /oauth2/authorization/github
    AUTH->>GH: OAuth 인증
    GH-->>AUTH: Access Token
    AUTH->>AUTH: User 생성 + Identity + JWT
    AUTH-->>W: 로그인 완료 (Bearer)

    par GitHub 프로필 수집 (비동기)
        AUTH->>GH: 리포·언어·커밋 수집
        GH-->>AUTH: 프로필 데이터
        AUTH->>AUTH: github_profiles UPSERT
        AUTH-->>W: SSE "GitHub 분석 완료"
    and 게스트 결과 이관
        W->>API: POST /onboarding/assessments/claim (guest_id)
        API->>ONB: 결과 이관 + AssessmentCompletedEvent
    end

    W->>API: POST /learning-paths/me/generate (SSE)
    API->>PATH: 경로 생성 오케스트레이션
    PATH->>AI: Claude Sonnet — 12주 로드맵 JSON
    AI-->>PATH: milestones[]
    PATH->>PATH: pgvector로 콘텐츠 매칭
    PATH-->>W: SSE "path_id: 1234, 이번주 과제 3개"
    W-->>U: 🎉 1st Aha — "당신의 3개월 백엔드 로드맵"
```

## 2. Sandbox 실습 → AI 코드 리뷰 (2nd Aha)

> **관련 화면**: SCR-W-SBX-001 (Sandbox), SCR-W-REV-001 (리뷰 결과)

```mermaid
sequenceDiagram
    participant U as 학습자
    participant W as Web (Monaco)
    participant API as API
    participant SBX as Sandbox Runner
    participant POOL as Docker Pool (gVisor)
    participant Kafka
    participant REV as AI Review Worker
    participant AI as AI Gateway (Claude)

    U->>W: 콘텐츠 끝 "실습 과제" 열기
    W->>API: GET /contents/{slug}/code-blocks/{blockId}
    API-->>W: starter_code + test_cases (숨김)

    U->>W: Monaco에서 코드 작성
    U->>W: "제출"
    W->>API: POST /sandbox/sessions {code, lang: java21}
    API->>SBX: 제출 처리
    SBX->>POOL: pre-warmed 컨테이너 할당 (java21-jdk)
    POOL-->>SBX: container_id
    SBX->>POOL: 코드 + 테스트 케이스 주입 → 실행 (30초 limit)
    POOL-->>SBX: stdout/stderr/exit_code
    SBX->>SBX: 테스트 결과 파싱
    SBX-->>W: SSE 실행 로그 + "2/3 통과"

    SBX->>Kafka: SandboxRunSubmittedEvent (Outbox)
    Kafka->>REV: 이벤트 소비
    REV->>AI: Claude Sonnet — 품질/보안/가독성 프롬프트
    AI-->>REV: {summary, strengths, improvements, security_issues}
    REV->>API: ai_code_reviews INSERT
    REV->>Kafka: AiReviewCompletedEvent

    W->>API: GET /ai-reviews/sandbox/{id} (polling or SSE)
    API-->>W: 리뷰 결과 표시
    W-->>U: "✅ 2/3 테스트 통과<br/>⚠️ line 42: LazyInitializationException 가능성"
```

## 3. AI 멘토 채팅 (컨텍스트 인식)

> **관련 화면**: SCR-W-MEN-001 (멘토 패널)

```mermaid
sequenceDiagram
    participant U as 학습자
    participant W as Web
    participant API as API
    participant MEN as Mentor Service
    participant AI as AI Gateway
    participant DB as PostgreSQL

    U->>W: "질문하기" 버튼
    W->>API: POST /ai-mentor/sessions
    API->>MEN: 세션 생성 + 컨텍스트 스냅샷
    MEN->>DB: 현재 콘텐츠 + 최근 5 Sandbox + 실패 테스트 조회
    DB-->>MEN: 컨텍스트 번들
    MEN->>MEN: context_snapshot JSON 저장
    MEN-->>W: session_id

    U->>W: "N+1이 왜 발생해?"
    W->>API: POST /messages (SSE)
    API->>MEN: 메시지 전달
    MEN->>AI: 프롬프트 = [System + Context + User]
    Note over MEN,AI: Context:<br/>- 현재 콘텐츠 "JPA 페치 전략"<br/>- 최근 과제: N+1 리팩토링 실패 1회<br/>- 실패 테스트: testFetchJoin
    AI-->>MEN: SSE 토큰 스트림
    MEN-->>W: SSE 전달
    W-->>U: "당신이 방금 푼 testFetchJoin을 보면..."

    MEN->>DB: 메시지 저장 + references
    MEN-->>W: done + references (content_id, sandbox_id)
    W-->>U: "참고 자료" 클릭 가능
```

## 4. 커뮤니티 피어 매칭

> **관련 화면**: SCR-W-COM-001 (Q&A), SCR-W-PATH-002 (이번 주 과제)

```mermaid
sequenceDiagram
    participant U as 학습자
    participant W as Web
    participant API as API
    participant PEER as Peer Matcher
    participant DB

    U->>W: "이번 주 과제" 화면
    W->>API: GET /community/peers/this-week
    API->>PEER: 피어 매칭 요청
    PEER->>DB: 같은 path_milestones.week_num + 개념 교집합 사용자 조회
    DB-->>PEER: 피어 23명
    PEER-->>W: "같은 주에 N+1 문제 푸는 사람 23명"

    U->>W: 피어 목록 클릭 → 질문 작성
    W->>API: POST /community/questions (concept_tags 자동 태깅)
    API->>DB: INSERT community_questions
    API->>PEER: 매칭된 피어에게 알림 (Outbox)
    API-->>W: question_id
```

## 5. 리텐션 트리거 (3일 미접속)

```mermaid
sequenceDiagram
    participant SCHED as Scheduler (Quartz)
    participant DB
    participant AI as AI Gateway
    participant NOTI as Notification
    participant U as 학습자
    participant M as Mobile/Email

    SCHED->>SCHED: 매일 09:00 (TZ별)
    SCHED->>DB: last_active_date > 3일 사용자 조회
    DB-->>SCHED: 이탈 위험 사용자 목록
    loop 각 사용자
        SCHED->>AI: 현재 경로 분석 + 난이도 재조정 제안
        AI-->>SCHED: "다음 과제가 너무 어려운 듯 — 기초 복습 제안"
        SCHED->>NOTI: StreakBreakingSoonEvent
        NOTI->>M: Push/Email "3일만에 돌아와요 — 쉬운 10분 복습부터"
        M-->>U: 알림
    end
```

## 6. OAuth2 + GitHub 프로필 수집 (상세)

```mermaid
sequenceDiagram
    participant U as 방문자
    participant W as Web
    participant SEC as Spring Security
    participant GH as GitHub OAuth
    participant GA as GitHub API
    participant Worker as GH Fetch Worker
    participant DB
    participant Kafka

    U->>W: "GitHub로 시작"
    W->>SEC: /oauth2/authorization/github
    SEC->>GH: redirect
    GH-->>U: 동의 화면
    U->>GH: 동의
    GH-->>SEC: /login/oauth2/code/github?code=...
    SEC->>GH: code → access_token 교환
    GH-->>SEC: access_token + user info

    SEC->>DB: user_oauth_identities 조회
    alt 최초 로그인
        SEC->>DB: User INSERT + user_oauth_identities INSERT
        SEC->>Kafka: UserRegisteredEvent (Outbox)
    else 기존 사용자
        SEC->>DB: 토큰 갱신
    end
    SEC-->>W: JWT 발급

    Kafka->>Worker: UserRegisteredEvent
    Worker->>GA: GET /user/repos (access_token)
    GA-->>Worker: 공개 리포 목록
    Worker->>GA: GET /user/events (최근 90일)
    Worker->>Worker: 언어 통계 집계
    Worker->>DB: github_profiles UPSERT + github_repositories 배치 insert
    Worker->>Kafka: GithubProfileFetchedEvent
    Kafka->>W: SSE 알림 "GitHub 분석 완료"
```
